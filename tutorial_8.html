1. Introduction

In previous weeks, we trained many models. Today we ask the harder question: how do we know they work? A model that looks great on training data but fails in production is worse than useless — it is dangerous.

Today's Goal

  • Diagnose overfitting using learning curves and train/validation gap analysis.
  • Build a complete confusion matrix visualization with TP, FP, FN, TN annotations.
  • Compute Precision, Recall, F1-Score, and AUC-ROC for multiple models.
  • Plot and compare ROC and Precision-Recall curves across three models.
  • Implement Stratified K-Fold cross-validation and interpret mean ± std results.
  • Run RandomizedSearchCV to find optimal hyperparameters.
  • Save the best model pipeline with joblib and deploy it as a Streamlit web app.

The four key evaluation pillars we will build today:

Pillar 1
Bias-Variance Diagnosis
Learning curves reveal whether a model is over- or under-fitting, and whether more data would help.
Pillar 2
Classification Metrics
Confusion matrix, Precision, Recall, F1 — because accuracy alone is always misleading for imbalanced classes.
Pillar 3
ROC & PR Curves
Threshold-independent ranking quality. AUC-ROC and Average Precision compare models fairly.
Pillar 4
Deployment
A model in a notebook helps no one. We save the full pipeline and wrap it in a Streamlit web application.

3. Creating Our Dataset — Customer Churn

We will build a synthetic customer churn dataset — a classic binary classification problem with class imbalance. Approximately 22% of customers churn, which is realistic and makes accuracy an unreliable metric.

Why synthetic data? We control the ground truth. We know exactly which features drive churn, so we can verify that our models are learning the right patterns and not just memorizing noise. 
np.random.seed(42)
n = 5000

# --- Generate realistic customer churn features ---
account_tenure   = np.random.exponential(scale=24, size=n).clip(1, 120)
monthly_spend    = np.random.lognormal(mean=4.5, sigma=0.8, size=n).clip(5, 500)
support_tickets  = np.random.poisson(lam=2, size=n).clip(0, 15)
login_frequency  = np.random.exponential(scale=8, size=n).clip(0.5, 50)
num_products     = np.random.randint(1, 8, size=n)
contract_type    = np.random.choice([0, 1, 2], size=n, p=[0.4, 0.35, 0.25])
payment_failures = np.random.poisson(lam=0.8, size=n).clip(0, 5)

# --- Churn probability --- long tenure, more products → less churn
churn_logit = (
    -0.05 * account_tenure
    - 0.002 * monthly_spend
    + 0.4  * support_tickets
    - 0.1  * login_frequency
    - 0.3  * num_products
    - 0.8  * contract_type
    + 0.9  * payment_failures
    + np.random.normal(0, 1.2, size=n)
)

churn_prob = 1 / (1 + np.exp(-churn_logit))
y = (churn_prob > 0.55).astype(int)   # ~22% churn rate

df = pd.DataFrame({
    'account_tenure':   account_tenure.round(1),
    'monthly_spend':    monthly_spend.round(2),
    'support_tickets':  support_tickets,
    'login_frequency':  login_frequency.round(1),
    'num_products':     num_products,
    'contract_type':    contract_type,
    'payment_failures': payment_failures,
    'churned':          y
})

feature_cols = [c for c in df.columns if c != 'churned']
X = df[feature_cols].values
y = df['churned'].values

print(f"Dataset shape:  {df.shape}")
print(f"Churn rate:     {y.mean():.1%}")
print(f"Non-churn (0):  {(y==0).sum()}")
print(f"Churn (1):      {(y==1).sum()}")
df[feature_cols].describe().round(2)

4. Step 1 — Train / Validation / Test Split

STEP 1 OF 11 • Protect Your Test Set
All Data (100%)
├── Train (60%) — model learns weights from this
├── Validation (20%) — used to tune hyperparameters
└── Test (20%) — final, one-time evaluation 
# Step 1: Hold out 20% as the sacred test set
X_temp, X_test, y_temp, y_test = train_test_split(
    X, y, test_size=0.20, random_state=42, stratify=y
)

# Step 2: From remaining 80%, reserve 25% (= 20% of total) as validation
X_train, X_val, y_train, y_val = train_test_split(
    X_temp, y_temp, test_size=0.25, random_state=42, stratify=y_temp
)

print(f"Total dataset:  {len(y):>6} samples")
print(f"  Train:        {len(y_train):>6} ({len(y_train)/len(y):.0%})")
print(f"  Validation:   {len(y_val):>6} ({len(y_val)/len(y):.0%})")
print(f"  Test:         {len(y_test):>6} ({len(y_test)/len(y):.0%})")
print(f"\nChurn rate in each split:")
print(f"  Train:      {y_train.mean():.1%}")
print(f"  Validation: {y_val.mean():.1%}")
print(f"  Test:       {y_test.mean():.1%}")
Verify stratification: All three splits should show nearly identical churn rates (~22%). If they differ significantly, your stratification failed. The stratify=y parameter in train_test_split guarantees this.

5. Step 2 — Build Three Model Pipelines

STEP 2 OF 11 • Prevent Data Leakage

We wrap each model in a Pipeline with StandardScaler. This is not just convenience — it is correctness. When the scaler is inside the pipeline and we use cross-validation, the scaler is refit on each training fold independently. Fitting the scaler before splitting is data leakage.

pipelines = {
    'Logistic Regression': Pipeline([
        ('scaler', StandardScaler()),
        ('clf', LogisticRegression(C=1.0, max_iter=1000, random_state=42))
    ]),
    'Random Forest': Pipeline([
        ('scaler', StandardScaler()),
        ('clf', RandomForestClassifier(n_estimators=100, max_depth=10, random_state=42))
    ]),
    'Gradient Boosting': Pipeline([
        ('scaler', StandardScaler()),
        ('clf', GradientBoostingClassifier(n_estimators=100, learning_rate=0.1,
                                           max_depth=4, random_state=42))
    ]),
}

6. Step 3 — Cross-Validation: Diagnosing Overfitting

STEP 3 OF 11 • Measure Generalization
cv = StratifiedKFold(n_splits=5, shuffle=True, random_state=42)
metrics = ['accuracy', 'f1_weighted', 'roc_auc']

cv_results = {}

print("=" * 65)
print(f"{'Model':<25} {'Metric':<18} {'Train':>8} {'Val':>8} {'Gap':>8}")
print("=" * 65)

for name, pipe in pipelines.items():
    result = cross_validate(
        pipe, X_train, y_train,
        cv=cv,
        scoring=metrics,
        return_train_score=True
    )
    cv_results[name] = result

    for m in metrics:
        tr = result[f'train_{m}'].mean()
        va = result[f'test_{m}'].mean()
        gap = tr - va
        flag = " ← overfit" if gap > 0.07 else ""
        print(f"{name:<25} {m:<18} {tr:>8.4f} {va:>8.4f} {gap:>+8.4f}{flag}")
    print("-" * 65)

7. Step 4 — Learning Curves: Does More Data Help?

STEP 4 OF 11 • Bias vs. Variance

Learning curves plot model performance as the training set size grows. They answer a critical question before you spend time collecting more data: would adding more samples actually improve my model?

fig, axes = plt.subplots(1, 3, figsize=(18, 5))
train_sizes_frac = np.linspace(0.1, 1.0, 10)

for ax, (name, pipe) in zip(axes, pipelines.items()):
    train_sizes, train_scores, val_scores = learning_curve(
        pipe, X_train, y_train,
        train_sizes=train_sizes_frac,
        cv=StratifiedKFold(n_splits=5, shuffle=True, random_state=42),
        scoring='f1_weighted',
        n_jobs=-1
    )

    tr_mean = train_scores.mean(axis=1)
    tr_std  = train_scores.std(axis=1)
    va_mean = val_scores.mean(axis=1)
    va_std  = val_scores.std(axis=1)

    ax.plot(train_sizes, tr_mean, 'b-o', label='Train', linewidth=2, markersize=5)
    ax.fill_between(train_sizes, tr_mean - tr_std, tr_mean + tr_std, alpha=0.1, color='blue')
    ax.plot(train_sizes, va_mean, 'r-o', label='Validation', linewidth=2, markersize=5)
    ax.fill_between(train_sizes, va_mean - va_std, va_mean + va_std, alpha=0.1, color='red')

    gap = tr_mean[-1] - va_mean[-1]
    ax.annotate(f'Gap: {gap:.3f}', xy=(train_sizes[-1], va_mean[-1]),
                xytext=(-60, -20), textcoords='offset points',
                fontsize=9, color='darkred',
                arrowprops=dict(arrowstyle='->', color='darkred'))

    ax.set_xlabel('Training Samples')
    ax.set_ylabel('F1-Score (Weighted)')
    ax.set_title(name, fontsize=11, fontweight='bold')
    ax.legend(fontsize=9)
    ax.set_ylim(0.5, 1.05)
    ax.spines['top'].set_visible(False)
    ax.spines['right'].set_visible(False)

plt.suptitle('Learning Curves — Does More Data Help?', fontsize=13, fontweight='bold')
plt.tight_layout()
plt.savefig('learning_curves.png', dpi=100, bbox_inches='tight')
plt.show()
How to read learning curves:
  • Both curves converge to a similar low value → Underfitting. More data will not help. Use a more complex model.
  • Train is high, validation is low, gap stays wide → Overfitting. More data will help. Also try regularization.
  • Both curves converge to a similar high value → well-fitted. Ready for production.

8. Step 5 — Confusion Matrix & Classification Report

STEP 5 OF 11 • Beyond Accuracy

Our churn dataset has ~22% positive class. A model that always predicts "no churn" would achieve 78% accuracy while catching zero churn cases. The confusion matrix reveals this failure immediately.

# Fit all models
fitted_models = {}
for name, pipe in pipelines.items():
    pipe.fit(X_train, y_train)
    fitted_models[name] = pipe

# Confusion matrices on VALIDATION set (not test yet!)
fig, axes = plt.subplots(1, 3, figsize=(16, 5))

for ax, (name, pipe) in zip(axes, fitted_models.items()):
    y_pred = pipe.predict(X_val)
    cm = confusion_matrix(y_val, y_pred)
    cm_norm = cm.astype(float) / cm.sum(axis=1)[:, np.newaxis]

    im = ax.imshow(cm_norm, interpolation='nearest', cmap='Blues', vmin=0, vmax=1)
    plt.colorbar(im, ax=ax, fraction=0.04)

    labels_text = [
        [f'TN\n{cm[0,0]}\n({cm_norm[0,0]:.0%})', f'FP\n{cm[0,1]}\n({cm_norm[0,1]:.0%})'],
        [f'FN\n{cm[1,0]}\n({cm_norm[1,0]:.0%})', f'TP\n{cm[1,1]}\n({cm_norm[1,1]:.0%})']
    ]
    for i in range(2):
        for j in range(2):
            color = 'white' if cm_norm[i, j] > 0.5 else 'black'
            ax.text(j, i, labels_text[i][j], ha='center', va='center',
                    fontsize=10, color=color, fontweight='bold')

    ax.set_xticks([0, 1])
    ax.set_yticks([0, 1])
    ax.set_xticklabels(['Predicted\nNo Churn', 'Predicted\nChurn'])
    ax.set_yticklabels(['Actual\nNo Churn', 'Actual\nChurn'])
    ax.set_title(name, fontsize=11, fontweight='bold')

plt.suptitle('Confusion Matrices (Validation Set)', fontsize=13, fontweight='bold')
plt.tight_layout()
plt.savefig('confusion_matrices.png', dpi=100, bbox_inches='tight')
plt.show()
# Print classification reports
for name, pipe in fitted_models.items():
    y_pred = pipe.predict(X_val)
    print(f"\n{'='*55}")
    print(f"  {name}")
    print(f"{'='*55}")
    print(classification_report(y_val, y_pred,
                                 target_names=['No Churn', 'Churn'],
                                 digits=4))
FN vs. FP — which error costs more?
A False Negative (predicted "no churn" but customer churned) = missed retention opportunity, lost revenue.
A False Positive (predicted "churn" but customer didn't) = unnecessary retention campaign cost.
If campaigns are cheap and churn is costly, maximize Recall. If campaigns are expensive, balance with F1.

ROC vs. PR — when to use which?
Use ROC-AUC when false positives and false negatives have similar cost and the dataset is roughly balanced.
Use PR-AUC (Average Precision) when the positive class is rare and highly important (fraud, disease). The PR curve is more informative in heavily imbalanced settings.

10. Step 7 — Threshold Optimization

STEP 7 OF 11 • 0.5 Is Rarely Optimal

The default threshold of 0.5 is almost never optimal for imbalanced problems. Let's find the threshold that maximizes F1-score on the validation set.

gb_pipe  = fitted_models['Gradient Boosting']
y_prob   = gb_pipe.predict_proba(X_val)[:, 1]

thresholds = np.arange(0.1, 0.9, 0.02)
f1_scores  = []
precision_scores = []
recall_scores    = []

from sklearn.metrics import precision_score, recall_score

for thresh in thresholds:
    y_pred_thresh = (y_prob >= thresh).astype(int)
    if y_pred_thresh.sum() == 0 or y_pred_thresh.sum() == len(y_pred_thresh):
        f1_scores.append(0); precision_scores.append(0); recall_scores.append(0)
        continue
    f1_scores.append(f1_score(y_val, y_pred_thresh, zero_division=0))
    precision_scores.append(precision_score(y_val, y_pred_thresh, zero_division=0))
    recall_scores.append(recall_score(y_val, y_pred_thresh, zero_division=0))

best_thresh = thresholds[np.argmax(f1_scores)]

fig, ax = plt.subplots(figsize=(11, 5))
ax.plot(thresholds, f1_scores,        'b-', linewidth=2.2, label='F1-Score')
ax.plot(thresholds, precision_scores, 'g--', linewidth=1.8, label='Precision')
ax.plot(thresholds, recall_scores,    'r--', linewidth=1.8, label='Recall')
ax.axvline(x=best_thresh, color='black', linestyle=':', linewidth=1.5,
           label=f'Best threshold = {best_thresh:.2f}')
ax.axvline(x=0.5, color='gray', linestyle=':', linewidth=1,
           label='Default threshold = 0.50')
ax.set_xlabel('Classification Threshold', fontsize=11)
ax.set_ylabel('Score', fontsize=11)
ax.set_title('Threshold Analysis — Gradient Boosting', fontsize=12, fontweight='bold')
ax.legend(fontsize=10)
ax.spines['top'].set_visible(False)
ax.spines['right'].set_visible(False)
plt.tight_layout()
plt.savefig('threshold_analysis.png', dpi=100, bbox_inches='tight')
plt.show()

print(f"--- Default threshold (0.50) ---")
y_pred_default = (y_prob >= 0.50).astype(int)
print(classification_report(y_val, y_pred_default, target_names=['No Churn', 'Churn'], digits=4))

print(f"--- Optimal threshold ({best_thresh:.2f}) ---")
y_pred_optimal = (y_prob >= best_thresh).astype(int)
print(classification_report(y_val, y_pred_optimal, target_names=['No Churn', 'Churn'], digits=4))

11. Step 8 — Hyperparameter Tuning with RandomizedSearchCV

STEP 8 OF 11 • Smart Search

Grid search tries every combination — this scales exponentially. Randomized search samples \(N\) random configurations and finds equally good results with a fraction of the compute. Research by Bergstra & Bengio (2012) confirmed this empirically.

from scipy.stats import randint, uniform

param_dist = {
    'clf__n_estimators':      randint(50, 400),
    'clf__learning_rate':     uniform(0.01, 0.3),
    'clf__max_depth':         randint(2, 8),
    'clf__min_samples_split': randint(2, 20),
    'clf__subsample':         uniform(0.6, 0.4),
}

gb_pipeline = Pipeline([
    ('scaler', StandardScaler()),
    ('clf', GradientBoostingClassifier(random_state=42))
])

random_search = RandomizedSearchCV(
    gb_pipeline,
    param_dist,
    n_iter=40,
    cv=StratifiedKFold(n_splits=5, shuffle=True, random_state=42),
    scoring='roc_auc',
    random_state=42,
    n_jobs=-1,
    verbose=0
)

print("Running RandomizedSearchCV (40 candidates × 5 folds = 200 fits)...")
random_search.fit(X_train, y_train)

print(f"\nBest parameters found:")
for k, v in random_search.best_params_.items():
    print(f"  {k:<35} {v}")
print(f"\nBest CV AUC-ROC:    {random_search.best_score_:.4f}")

best_model = random_search.best_estimator_
y_prob_best = best_model.predict_proba(X_val)[:, 1]
val_auc = auc(*roc_curve(y_val, y_prob_best)[:2])
print(f"Validation AUC-ROC: {val_auc:.4f}")
# Visualize search results
results_df = pd.DataFrame(random_search.cv_results_)
results_df = results_df[['param_clf__n_estimators', 'param_clf__learning_rate',

results_df.columns = ['n_estimators', 'learning_rate', 'max_depth', 'mean_auc', 'std_auc']
results_df = results_df.sort_values('mean_auc', ascending=False)

12. Step 9 — The Final Test Evaluation (Once Only!)

STEP 9 OF 11 • The Moment of Truth
This is the moment of truth. All decisions — model choice, hyperparameters, threshold — were made using only train and validation data. We now evaluate exactly once on the held-out test set. If you evaluate and then adjust, you have leaked information and your reported metrics are dishonest. 
final_model = random_search.best_estimator_
final_model.fit(X_train, y_train)   # refit on full training set

y_test_prob = final_model.predict_proba(X_test)[:, 1]
y_test_pred_optimal = (y_test_prob >= best_thresh).astype(int)

print("=" * 55)
print("  FINAL TEST SET EVALUATION (UNSEEN DATA)")
print("=" * 55)

fpr_t, tpr_t, _ = roc_curve(y_test, y_test_prob)
test_auc = auc(fpr_t, tpr_t)
print(f"\nAUC-ROC (test):  {test_auc:.4f}")
print(f"AUC-ROC (val):   {val_auc:.4f}")
print(f"Gap (overfit?):  {val_auc - test_auc:+.4f}")

print(f"\n--- Classification Report (threshold = {best_thresh:.2f}) ---")
print(classification_report(y_test, y_test_pred_optimal,
                             target_names=['No Churn', 'Churn'], digits=4))

    print(f"  Customer {i+1}: {label} (p={prob:.3f})")
Always save the full Pipeline, not just the model. If you save only the classifier, you must manually reapply the same scaler at inference time — a maintenance nightmare. The joblib file bundles the scaler and classifier together.

15. Experiment Ideas
