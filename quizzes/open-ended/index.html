Easy Questions — 1 pt each
Q1 Easy  ·  1 pt  ·  Week 5
What range of values does the sigmoid function output? Why is this useful for classification?
Model Answer The sigmoid outputs values in (0, 1). This is useful because the output can be interpreted as a probability — e.g., 0.85 means 85% chance of being class 1. Values are then thresholded (usually at 0.5) to make a final class prediction.
Q2 Easy  ·  1 pt  ·  Week 3
Name the four types of data and give one example of each.
Model Answer Nominal — categories with no order (e.g., City: Baku, Ganja).
Ordinal — ordered categories, no precise numeric gap (e.g., Grade: Poor / Fair / Good).
Discrete — countable integers (e.g., number of students in a class).
Continuous — any real number within a range (e.g., temperature: 36.6°C).
Q3 Easy  ·  1 pt  ·  Week 4
What is Mean Squared Error (MSE)? Write the formula and explain in one sentence what it measures.
Model Answer MSE = (1/n) × Σ(yᵢ − ŷᵢ)². It measures the average squared difference between the true values and the model's predictions — larger errors are penalized more heavily due to squaring.
Q4 Easy  ·  1 pt  ·  Week 5
What are "support vectors" in an SVM, and what role do they play?
Model Answer Support vectors are the training data points closest to the decision boundary. They define the margin — the gap between the two classes. Only these points matter for determining the boundary; removing any other point does not change the model.
Medium Questions — 2 pts each
Q5 Medium  ·  2 pts  ·  Week 3
A dataset has a column "Membership Level" with values: Bronze, Silver, Gold, Platinum.
(a) What data type is this column?
(b) Can you apply One-Hot Encoding to it? Should you? Explain.
Model Answer (a) Ordinal — the categories have a meaningful order (Bronze < Silver < Gold < Platinum).
(b) You can apply One-Hot Encoding, but you should not, because it would destroy the order information. A better approach is ordinal encoding (e.g., Bronze=1, Silver=2, Gold=3, Platinum=4), which preserves the ranking the model can learn from.
Q6 Medium  ·  2 pts  ·  Week 4
Explain the difference between overfitting and underfitting. For each, describe what you would observe in training vs. test error, and give one way to fix it.
Model Answer Overfitting: The model learns the training data too well, including noise. Training error is very low, test error is much higher. Fix: use more training data, simplify the model, or add regularization.
Underfitting: The model is too simple to capture the pattern. Both training and test errors are high. Fix: use a more complex model or add more relevant features.
Q7 Medium  ·  2 pts  ·  Week 5
A cancer detection model achieves 99% accuracy on a dataset where 99% of samples are healthy.
(a) Is this model useful? Explain.
(b) Which metric should be used instead, and why?
Model Answer (a) No — a model that always predicts "healthy" would also get 99% accuracy without detecting a single cancer case. The accuracy is misleading on this imbalanced dataset.
(b) Recall should be prioritized. Recall = TP / (TP + FN) measures how many actual cancer cases the model catches. Missing a cancer case (false negative) is far more dangerous than a false alarm.
Q8 Medium  ·  2 pts  ·  Week 3 & 4
A column "Age" has values [22, 25, 30, 28, NaN, 24].
(a) Describe two different strategies to handle the missing value.
(b) Which one would you choose and why?
Model Answer (a) Strategy 1: Drop the row — remove the record with the missing value. Strategy 2: Mean imputation — replace NaN with the mean age ((22+25+30+28+24)/5 = 25.8).
(b) Mean imputation is better here because: the dataset is small (dropping a row loses 20% of data), and age values are close together so the mean is a reasonable estimate.
Hard Questions — 4 pts each
Q9 Hard  ·  4 pts  ·  Week 4 & 5
A linear regression model is trained to predict house prices and gives R² = 0.92 on training data but R² = 0.43 on test data.
(a) What problem does this indicate?
(b) The team proposes switching to a polynomial regression of degree 10. Will this help? Justify your answer.
(c) Suggest a better solution.
Model Answer (a) Overfitting — the model fits training data very well but fails to generalize.
(b) No — degree-10 polynomial adds far more complexity, which will increase overfitting and make the test performance even worse. High-degree polynomials memorize noise rather than learning the pattern.
(c) Better solutions: use a simpler model (lower-degree polynomial or regularization), gather more training data, or apply feature selection to remove irrelevant predictors.
Q10 Hard  ·  4 pts  ·  Week 3, 4 & 5
You are building a model to classify emails as Spam / Not Spam. Your raw dataset has the following issues:
  • Column "Category" — values: Work, Personal, Promotions, Social
  • Column "Subject Length" — values range from 2 to 300 characters
  • Column "Sender Reputation Score" — ranges from 0 to 1000, with some values at 5000 (outliers)
  • Column "Days Since Last Email" — 35% values missing
For each column, state what preprocessing step you would apply and briefly explain why.
Model Answer Category: One-Hot Encoding — it is nominal (no order), so encode each value as a binary column. Avoid ordinal encoding which would impose a false order.
Subject Length: Min-Max Normalization — values are bounded and no major outliers. Scale to [0, 1] so the feature doesn't dominate gradient-based training.
Sender Reputation Score: Z-Score Standardization — the outliers (5000) would compress all other values into a tiny range with min-max. Z-score handles outliers more gracefully.
Days Since Last Email: Impute with median (robust to skew) or mean, since 35% is significant but not high enough to drop the entire column. Optionally add an indicator column (is_missing = 1/0) to let the model learn from the fact that the value was absent.
