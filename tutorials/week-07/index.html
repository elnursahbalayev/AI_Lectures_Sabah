1. Introduction

Last week, we classified fraudulent transactions using labeled data. Today, we have no labels. We are given a dataset of customer behavior and must discover natural groupings — without being told what those groups should be.

Today's Goal

  • Build a clean preprocessing pipeline with standardization and PCA.
  • Use the elbow method and silhouette analysis to pick the optimal \(K\).
  • Train K-Means and visualize the clusters in 2D PCA space.
  • Apply DBSCAN and compare results to K-Means.
  • Detect anomalous customers using Isolation Forest.
  • Profile each cluster to understand who lives inside each segment.

The 4 latent customer segments we'll try to discover:

Segment 0
🏆 High-Value Loyals
High spend, high frequency, long account tenure. The most valuable segment — reward & upsell.
Segment 1
💰 Bargain Hunters
Low order value, high frequency, short tenure. Price-sensitive but engaged — target with promotions.
Segment 2
⚠️ Dormant At-Risk
Medium spend, very low frequency, long since last purchase. Launch re-engagement campaigns.
Segment 3
🚀 New High-Spenders
High spend but low frequency and new accounts. Nurture to convert into loyal customers.

3. Creating Our Dataset

We'll generate a synthetic dataset of 2,000 e-commerce customers with 5 behavioral features and 4 latent segments. The segment_gt column is our ground truth — but we will not use it during training.

# ============================================================
# GENERATE SYNTHETIC CUSTOMER DATASET
# 2,000 customers across 4 behavioral segments.
# We know the true labels, but we will HIDE them during training
# to simulate a real unsupervised problem.
# ============================================================

np.random.seed(42)
n = 2000

# Segment 0: High-Value Loyals   (500 customers)
# Segment 1: Bargain Hunters      (600 customers)
# Segment 2: Dormant At-Risk      (550 customers)
# Segment 3: New High-Spenders    (350 customers)
segment_sizes = [500, 600, 550, 350]

def make_segment(size, spend_mean, freq_mean, tenure_mean,
                 recency_mean, categories_mean):
    return {
        'avg_order_value': np.random.normal(
            spend_mean, spend_mean * 0.2, size).clip(5, 500),
        'purchase_frequency': np.random.normal(
            freq_mean, freq_mean * 0.3, size).clip(0.5, 50),
        'account_tenure_months': np.random.normal(
            tenure_mean, tenure_mean * 0.25, size).clip(1, 60),
        'days_since_last_purchase': np.random.normal(
            recency_mean, recency_mean * 0.3, size).clip(1, 365),
        'num_categories_purchased': np.random.normal(
            categories_mean, 1.5, size).clip(1, 15).astype(int),
    }

segments_data = [
    make_segment(segment_sizes[0], spend_mean=180, freq_mean=12,
                 tenure_mean=36, recency_mean=15,  categories_mean=8),
    make_segment(segment_sizes[1], spend_mean=45,  freq_mean=20,
                 tenure_mean=12, recency_mean=10,  categories_mean=4),
    make_segment(segment_sizes[2], spend_mean=95,  freq_mean=2,
                 tenure_mean=42, recency_mean=150, categories_mean=3),
    make_segment(segment_sizes[3], spend_mean=220, freq_mean=3,
                 tenure_mean=5,  recency_mean=20,  categories_mean=6),
]

data = {}
for key in segments_data[0].keys():
    data[key] = np.concatenate([seg[key] for seg in segments_data])

df = pd.DataFrame(data)
df['segment_gt'] = np.concatenate([
    np.full(sz, i) for i, sz in enumerate(segment_sizes)
])  # Ground-truth labels — DO NOT use in training

df = df.sample(frac=1, random_state=42).reset_index(drop=True)

feature_cols = ['avg_order_value', 'purchase_frequency',
                'account_tenure_months', 'days_since_last_purchase',
                'num_categories_purchased']
X = df[feature_cols].values

print(f"Dataset shape: {df.shape}")
print(f"\nSegment distribution:")
print(df['segment_gt'].value_counts().sort_index())
print(f"\nFirst 5 rows:")
df[feature_cols].head()

5. Step 2 — Preprocessing: Standardization & PCA

STEP 2 OF 8 • Scale and Compress

5.1 Why Standardize?

K-Means uses Euclidean distance. Without standardization, features with large scales (e.g., days_since_last_purchase, range 1–365) dominate over small-scale features (num_categories_purchased, range 1–15).

# ============================================================
# STANDARDIZE FEATURES
# After scaling: mean = 0, std = 1 for every feature.
# This ensures no single feature dominates distance calculations.
# ============================================================

scaler = StandardScaler()
X_scaled = scaler.fit_transform(X)

print("Before scaling:")
print(f"  avg_order_value:          mean={X[:, 0].mean():.1f}, std={X[:, 0].std():.1f}")
print(f"  days_since_last_purchase: mean={X[:, 3].mean():.1f}, std={X[:, 3].std():.1f}")
print("\nAfter scaling:")
print(f"  avg_order_value:          mean={X_scaled[:, 0].mean():.1f}, std={X_scaled[:, 0].std():.1f}")
print(f"  days_since_last_purchase: mean={X_scaled[:, 3].mean():.1f}, std={X_scaled[:, 3].std():.1f}")

5.2 PCA — Variance Analysis

# ============================================================
# PCA VARIANCE ANALYSIS
# How many components do we need to retain 95% of information?
# The scree plot shows individual component variance.
# The cumulative plot helps select the right number.
# ============================================================

pca_full = PCA()
pca_full.fit(X_scaled)

cumvar = np.cumsum(pca_full.explained_variance_ratio_)

fig, axes = plt.subplots(1, 2, figsize=(14, 5))

# Plot 1: Individual variance explained
axes[0].bar(range(1, len(pca_full.explained_variance_ratio_) + 1),
            pca_full.explained_variance_ratio_, color='#3b82f6', alpha=0.8)
axes[0].set_xlabel('Principal Component')
axes[0].set_ylabel('Variance Explained')
axes[0].set_title('Variance Explained per Component')

# Plot 2: Cumulative variance explained
axes[1].plot(range(1, len(cumvar) + 1), cumvar, 'bo-', markersize=8)
axes[1].axhline(y=0.95, color='red', linestyle='--', label='95% threshold')
axes[1].set_xlabel('Number of Components')
axes[1].set_ylabel('Cumulative Variance Explained')
axes[1].set_title('Cumulative Variance Explained')
axes[1].legend()
axes[1].set_ylim(0, 1.05)

plt.suptitle('PCA Variance Analysis', fontsize=14, fontweight='bold')
plt.tight_layout()
plt.savefig('pca_variance.png', dpi=100, bbox_inches='tight')
plt.show()

n_for_95 = np.argmax(cumvar >= 0.95) + 1
print(f"Components needed for 95% variance: {n_for_95}")
print(f"Per-component variance: {pca_full.explained_variance_ratio_.round(3)}")
# ============================================================
# REDUCE TO 2D FOR CLUSTERING AND VISUALIZATION
# 2 components allow us to plot every point on a scatter chart.
# We check how much variance is retained.
# ============================================================

pca_2d = PCA(n_components=2)
X_pca = pca_2d.fit_transform(X_scaled)

print(f"Original dimensions: {X_scaled.shape[1]}")
print(f"Reduced dimensions:  {X_pca.shape[1]}")
print(f"Total variance retained: {pca_2d.explained_variance_ratio_.sum():.2%}")
Important: Always fit the scaler and PCA on training data only, then transform test data. Here we have no predefined train/test split (unsupervised), so we fit on the full dataset — which is standard practice for clustering.

6. Step 3 — Finding the Optimal K

STEP 3 OF 8 • Elbow Method + Silhouette

6.1 Elbow Method

# ============================================================
# ELBOW METHOD
# We run K-Means for K=1 to K=10 and plot WCSS (inertia).
# The "elbow" is where inertia stops dropping sharply.
# This is a heuristic — a tool to narrow down candidates.
# ============================================================

inertias = []
K_range = range(1, 11)

for k in K_range:
    km = KMeans(n_clusters=k, init='k-means++',
                n_init=10, max_iter=300, random_state=42)
    km.fit(X_pca)
    inertias.append(km.inertia_)

plt.figure(figsize=(9, 5))
plt.plot(K_range, inertias, 'bo-', markersize=9, linewidth=2)
for k, inertia in zip(K_range, inertias):
    plt.annotate(f'{inertia:.0f}', (k, inertia),
                 textcoords='offset points', xytext=(8, 5), fontsize=8)
plt.xlabel('Number of Clusters K')
plt.ylabel('Inertia (WCSS)')
plt.title('Elbow Method for Optimal K')
plt.xticks(K_range)
plt.tight_layout()
plt.savefig('elbow_method.png', dpi=100, bbox_inches='tight')
plt.show()

6.2 Silhouette Analysis

# ============================================================
# SILHOUETTE ANALYSIS
# For each K, silhouette_score measures how well-separated
# the clusters are. Higher = better.
# s(i) = (b(i) - a(i)) / max(a(i), b(i))
#   a(i) = avg intra-cluster distance
#   b(i) = avg distance to nearest other cluster
# ============================================================

silhouette_scores = []

for k in range(2, 11):
    km = KMeans(n_clusters=k, init='k-means++', n_init=10, random_state=42)
    labels = km.fit_predict(X_pca)
    score = silhouette_score(X_pca, labels)
    silhouette_scores.append(score)
    print(f"K={k:2d}: Silhouette Score = {score:.4f}")

# Select the K with highest silhouette score
best_k = np.argmax(silhouette_scores) + 2   # +2 because range started at K=2

plt.figure(figsize=(9, 5))
plt.plot(range(2, 11), silhouette_scores, 'go-', markersize=9, linewidth=2)
plt.axvline(x=best_k, color='red', linestyle='--', label=f'Best K={best_k}')
plt.xlabel('Number of Clusters K')
plt.ylabel('Silhouette Score')
plt.title('Silhouette Score vs. K')
plt.legend()
plt.xticks(range(2, 11))
plt.tight_layout()
plt.savefig('silhouette_scores.png', dpi=100, bbox_inches='tight')
plt.show()

print(f"\n✓ Optimal K by Silhouette: {best_k}")
Elbow vs. Silhouette: The elbow is visual and subjective. The silhouette score is quantitative. Use the elbow to narrow down candidates (e.g., K=3 to K=5), then use the silhouette score to make the final decision.

7. Step 4 — Train K-Means

STEP 4 OF 8 • Fit & Visualize Clusters

7.1 Fit the Final K-Means Model

# ============================================================
# FIT FINAL K-MEANS MODEL
# n_init=10: run 10 times with different centroid seeds,
#            keep the result with the lowest inertia.
# ============================================================

km_final = KMeans(n_clusters=best_k, init='k-means++',

    count = (cluster_labels == k).sum()
    print(f"  Cluster {k}: {count} customers ({count/n:.1%})")

7.2 Visualize K-Means Clusters

colors = ['#3b82f6', '#ef4444', '#4ade80', '#f59e0b', '#8b5cf6', '#06b6d4']

# Plot centroids
centroids_pca = km_final.cluster_centers_
ax.scatter(centroids_pca[:, 0], centroids_pca[:, 1],

for k, (cx, cy) in enumerate(centroids_pca):
    ax.annotate(f'C{k}', (cx, cy),
                textcoords='offset points', xytext=(8, 8),
                fontsize=12, fontweight='bold')

ax.set_xlabel(f"PC1 ({pca_2d.explained_variance_ratio_[0]:.1%} variance)", fontsize=11)
ax.set_ylabel(f"PC2 ({pca_2d.explained_variance_ratio_[1]:.1%} variance)", fontsize=11)
ax.set_title(f'K-Means Customer Clusters (K={best_k}) — PCA 2D Projection', fontsize=13)
ax.legend(fontsize=10)

7.3 Deep Silhouette Plot

# ============================================================
# SILHOUETTE PLOT — Per-Cluster Quality Visualization
# Each horizontal bar shows the silhouette coefficient for
# one data point. Bars extending left of zero (red dashed line)
# are likely misclassified. All bars should be similar in width
# for a well-balanced clustering.
# ============================================================

from sklearn.metrics import silhouette_samples
import matplotlib.cm as cm

fig, ax = plt.subplots(figsize=(10, 7))
sample_silhouette_values = silhouette_samples(X_pca, cluster_labels)

y_lower = 10
for k in range(best_k):
    ith_values = sample_silhouette_values[cluster_labels == k]
    ith_values.sort()
    size_k = ith_values.shape[0]
    y_upper = y_lower + size_k

    color = cm.nipy_spectral(float(k) / best_k)
    ax.fill_betweenx(np.arange(y_lower, y_upper), 0, ith_values,
                     facecolor=color, edgecolor=color, alpha=0.7)
    ax.text(-0.05, y_lower + 0.5 * size_k, str(k))
    y_lower = y_upper + 10

avg_score = silhouette_score(X_pca, cluster_labels)
ax.axvline(x=avg_score, color="red", linestyle="--",
           label=f'Mean: {avg_score:.3f}')
ax.set_xlabel("Silhouette Coefficient")
ax.set_ylabel("Cluster")
ax.set_title(f"Silhouette Plot for K={best_k}")
ax.legend()
plt.tight_layout()
plt.savefig('silhouette_plot.png', dpi=100, bbox_inches='tight')
plt.show()

8. Step 5 — DBSCAN Clustering

STEP 5 OF 8 • Density-Based Alternative
# ============================================================
# DBSCAN — DENSITY-BASED CLUSTERING
# DBSCAN does not require K. It finds clusters as dense regions.
# Points that don't belong to any dense region are labeled -1
# (noise). These are our outlier customers.
#
# Tuning guide:
#   eps:         start around 0.3-0.8 on PCA-reduced data
#   min_samples: rule of thumb → 2 * n_features
# ============================================================

dbscan = DBSCAN(eps=0.5, min_samples=10)
labels_db = dbscan.fit_predict(X_pca)

n_clusters_db = len(set(labels_db)) - (1 if -1 in labels_db else 0)
n_noise_db    = (labels_db == -1).sum()

print(f"DBSCAN Results:")
print(f"  Estimated clusters: {n_clusters_db}")
print(f"  Noise points:       {n_noise_db} ({n_noise_db/n:.1%})")

if n_clusters_db >= 2:
    db_mask = labels_db != -1
    db_score = silhouette_score(X_pca[db_mask], labels_db[db_mask])
    print(f"  Silhouette (excl. noise): {db_score:.4f}")
# ============================================================
# COMPARE K-MEANS vs. DBSCAN SIDE BY SIDE
# Left: K-Means clean spherical clusters
# Right: DBSCAN with noise points marked as black X
# ============================================================

fig, axes = plt.subplots(1, 2, figsize=(16, 6))

# Left: K-Means
for k in range(best_k):
    mask = cluster_labels == k
    axes[0].scatter(X_pca[mask, 0], X_pca[mask, 1],
                    c=colors[k % len(colors)], label=f'Cluster {k}',
                    alpha=0.6, s=30)
axes[0].set_title(f'K-Means (K={best_k})', fontsize=12)
axes[0].set_xlabel('PC1')
axes[0].set_ylabel('PC2')
axes[0].legend(fontsize=9)

# Right: DBSCAN
for label in sorted(set(labels_db)):
    mask = labels_db == label
    if label == -1:
        axes[1].scatter(X_pca[mask, 0], X_pca[mask, 1],
                        c='black', marker='x', s=30,
                        label='Noise', alpha=0.5)
    else:
        axes[1].scatter(X_pca[mask, 0], X_pca[mask, 1],
                        c=colors[label % len(colors)],
                        label=f'Cluster {label}', alpha=0.6, s=30)

axes[1].set_title(
    f'DBSCAN (eps=0.5, min_samples=10)\n{n_clusters_db} clusters, {n_noise_db} noise',
    fontsize=12)
axes[1].set_xlabel('PC1')
axes[1].set_ylabel('PC2')
axes[1].legend(fontsize=9)

plt.suptitle('K-Means vs. DBSCAN Comparison', fontsize=13, fontweight='bold')
plt.tight_layout()
plt.savefig('clustering_comparison.png', dpi=100, bbox_inches='tight')
plt.show()
Discussion: How do the cluster shapes differ? Which points did DBSCAN label as noise? Are these customers on the boundaries between segments? Try changing eps to 0.3, 0.8, 1.2 and observe how the number of noise points and clusters changes.

9. Step 6 — Anomaly Detection with Isolation Forest

STEP 6 OF 8 • Find the Outliers
# ============================================================
# ISOLATION FOREST — ANOMALY DETECTION
# Anomalies are easier to isolate: they need fewer random splits
# to separate from the rest of the data.
# contamination: the expected fraction of anomalies in the dataset
# ============================================================

iso_forest = IsolationForest(
    n_estimators=200,
    contamination=0.03,   # expect ~3% unusual customers
    random_state=42
)
anomaly_labels = iso_forest.fit_predict(X_scaled)   # fit on original scaled space
anomaly_scores = iso_forest.score_samples(X_scaled) # lower = more anomalous

n_anomalies = (anomaly_labels == -1).sum()
print(f"Isolation Forest — Anomaly Detection")
print(f"  Total customers:    {n}")
print(f"  Anomalies detected: {n_anomalies} ({n_anomalies/n:.1%})")
# ============================================================
# VISUALIZE ANOMALIES ON PCA 2D PROJECTION
# Normal customers: blue dots
# Anomalous customers: red stars — investigate these!
# ============================================================

fig, ax = plt.subplots(figsize=(11, 7))

normal_mask  = anomaly_labels == 1
anomaly_mask = anomaly_labels == -1

ax.scatter(X_pca[normal_mask, 0], X_pca[normal_mask, 1],
           c='#3b82f6', alpha=0.5, s=30,
           label='Normal', edgecolors='none')
ax.scatter(X_pca[anomaly_mask, 0], X_pca[anomaly_mask, 1],
           c='#ef4444', marker='*', s=150,
           label=f'Anomaly ({n_anomalies})',
           edgecolors='black', linewidth=0.5, zorder=5)

ax.set_xlabel(f"PC1 ({pca_2d.explained_variance_ratio_[0]:.1%} variance)")
ax.set_ylabel(f"PC2 ({pca_2d.explained_variance_ratio_[1]:.1%} variance)")
ax.set_title('Isolation Forest — Anomaly Detection on Customer Data')
ax.legend(fontsize=11)
plt.tight_layout()
plt.savefig('anomaly_detection.png', dpi=100, bbox_inches='tight')
plt.show()

# Profile anomalies vs. normal customers
df_anom   = df[feature_cols][anomaly_mask]
df_normal = df[feature_cols][normal_mask]

print("\nAnomaly vs. Normal profile:")
comparison = pd.DataFrame({
    'Normal Mean':  df_normal.mean(),
    'Anomaly Mean': df_anom.mean(),
}).round(2)
print(comparison)
Business interpretation: Anomalous customers may represent accounts to investigate (potential bots, data errors, or genuinely unusual customers worth personal outreach). Their feature profile tells you why they're anomalous.

10. Step 7 — Cluster Profiling & Business Insights

STEP 7 OF 8 • Turn Numbers into Strategy

This is the most important step: turning cluster labels into actionable business knowledge.

# ============================================================
# CLUSTER PROFILE TABLE
# For each cluster, compute the mean of each original feature.
# This tells you: "What does a typical customer in Cluster 0 look like?"
# ============================================================

df['cluster'] = cluster_labels

cluster_profiles = df.groupby('cluster')[feature_cols].mean().round(2)
print("=" * 70)
print("CLUSTER PROFILES — Mean Feature Values per Cluster")
print("=" * 70)
print(cluster_profiles.to_string())
# ============================================================
# CLUSTER PROFILE BAR CHARTS
# Normalize each feature to [0,1] for visual comparison.

fig, axes = plt.subplots(1, best_k, figsize=(5 * best_k, 5), sharey=False)

profile_normalized = (
    (cluster_profiles - cluster_profiles.min()) /

for k, ax in enumerate(axes.flatten()):

    ax.spines['top'].set_visible(False)
    ax.spines['right'].set_visible(False)
    for bar, val in zip(bars, cluster_profiles.loc[k]):

plt.suptitle('Customer Cluster Profiles (Normalized)', fontsize=13, fontweight='bold')
