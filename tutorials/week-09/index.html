1. Introduction

If you use a framework without understanding backpropagation, you are a mechanic who can only fix cars by replacing the engine. Build it once from scratch — then you can debug anything.

Today we implement a complete neural network using only NumPy. No PyTorch, no TensorFlow, no Scikit-learn. Every matrix multiply, every gradient, every weight update is written by you. By the end of this session, your handcrafted network will classify handwritten digits with over 95% accuracy.

Today's Goal

  • Load and explore the MNIST dataset — 70,000 grayscale digit images.
  • Preprocess: flatten, normalize, and one-hot encode.
  • Implement relu(), softmax(), and cross_entropy_loss() from scratch.
  • Build forward() using matrix operations for a [784 → 256 → 128 → 10] MLP.
  • Derive and implement backward() using the Chain Rule.
  • Train with mini-batch gradient descent and track loss / accuracy.
  • Visualize training curves and a confusion matrix over 10 digit classes.
  • Inspect misclassified examples and understand where the model fails.

5. Step 3 — Implement Activation Functions and Loss

STEP 3 OF 10 • Core Math Functions
def relu(z):
    """Rectified Linear Unit activation."""
    return np.maximum(0, z)

def relu_derivative(z):
    """Derivative of ReLU: 1 where z > 0, 0 elsewhere."""
    return (z > 0).astype(float)

def softmax(z):
    """
    Numerically stable softmax over the last axis.
    z: (batch_size, n_classes)
    """
    z_shifted = z - z.max(axis=1, keepdims=True)   # subtract row-wise max
    exp_z = np.exp(z_shifted)
    return exp_z / exp_z.sum(axis=1, keepdims=True)

def cross_entropy_loss(y_hat, y_true):
    """
    Categorical cross-entropy loss.
    y_hat:  (n, K) — softmax probabilities
    y_true: (n,)   — integer class labels
    """
    n = len(y_true)
    log_probs = -np.log(y_hat[np.arange(n), y_true] + 1e-9)
    return log_probs.mean()

def accuracy(y_hat, y_true):
    """Fraction of correct predictions."""
    return (np.argmax(y_hat, axis=1) == y_true).mean()

6. Step 4 — Weight Initialization

STEP 4 OF 10 • He Initialization
def initialize_weights(layer_sizes):
    """
    He initialization for a network with ReLU activations.
    layer_sizes: list of layer widths, e.g. [784, 256, 128, 10]
    Returns a dict of weight matrices and bias vectors.
    """
    params = {}
    L = len(layer_sizes) - 1

    for l in range(1, L + 1):
        n_in  = layer_sizes[l - 1]
        n_out = layer_sizes[l]

        params[f'W{l}'] = np.random.randn(n_out, n_in) * np.sqrt(2.0 / n_in)
        params[f'b{l}'] = np.zeros((n_out, 1))

    return params

# --- Define our MLP architecture ---
LAYER_SIZES = [784, 256, 128, 10]

params = initialize_weights(LAYER_SIZES)

total_params = 0
for l in range(1, len(LAYER_SIZES)):
    W_shape = params[f'W{l}'].shape
    b_shape = params[f'b{l}'].shape
    n = W_shape[0] * W_shape[1] + b_shape[0]
    total_params += n
    print(f"Layer {l}: W{l} {W_shape}  b{l} {b_shape}  → {n:,} parameters")

print(f"\nTotal trainable parameters: {total_params:,}")
He initialization prevents vanishing/exploding activations. If weights are too small, activations shrink to zero through layers. If too large, they explode. He scaling (\(\sqrt{2/n_{in}}\)) is calibrated so ReLU activations maintain unit variance across depth.

7. Step 5 — The Forward Pass

STEP 5 OF 10 • Input → Prediction
def forward_pass(X, params):
    """
    Full forward pass for [784 → 256 → 128 → 10] MLP.
    X: (batch_size, 784) — input batch
    Returns: y_hat (n, 10), cache dict for backprop
    """
    cache = {}
    cache['A0'] = X                                 # (n, 784)

    # Hidden layer 1: ReLU
    Z1 = cache['A0'] @ params['W1'].T + params['b1'].T
    A1 = relu(Z1)
    cache['Z1'], cache['A1'] = Z1, A1               # (n, 256)

    # Hidden layer 2: ReLU
    Z2 = A1 @ params['W2'].T + params['b2'].T
    A2 = relu(Z2)
    cache['Z2'], cache['A2'] = Z2, A2               # (n, 128)

    # Output layer: Softmax
    Z3 = A2 @ params['W3'].T + params['b3'].T
    Y_hat = softmax(Z3)
    cache['Z3'], cache['Y_hat'] = Z3, Y_hat         # (n, 10)

    return Y_hat, cache

# --- Test forward pass with one batch ---
X_batch = X_train[:64]
y_hat_test, cache_test = forward_pass(X_batch, params)

print(f"Input shape:  {X_batch.shape}")
print(f"Output shape: {y_hat_test.shape}")
print(f"Sample output (first image): {y_hat_test[0].round(4)}")

print(f"Initial loss (should be ~ln(10) ≈ 2.303): {cross_entropy_loss(y_hat_test, y_train[:64]):.4f}")

8. Step 6 — Backpropagation

STEP 6 OF 10 • Chain Rule Layer by Layer

We now implement the backward pass — computing gradients \(\partial \mathcal{L}/\partial W\) for every layer using the Chain Rule. Work backwards from the output layer to the input.

def backward_pass(y_true, cache, params):
    """
    Backward pass: compute gradients for all parameters.
    y_true: (n,) integer labels
    Returns: grads dict with dW1, db1, ..., dW3, db3
    """
    n = len(y_true)
    grads = {}

    Y_hat = cache['Y_hat']      # (n, 10)
    A2    = cache['A2']         # (n, 128)
    A1    = cache['A1']         # (n, 256)
    A0    = cache['A0']         # (n, 784)
    Z2    = cache['Z2']         # (n, 128)
    Z1    = cache['Z1']         # (n, 256)

    # One-hot encode ground truth
    Y_onehot = np.zeros_like(Y_hat)            # (n, 10)
    Y_onehot[np.arange(n), y_true] = 1.0

    # --- Output layer (softmax + cross-entropy gradient) ---
    dZ3 = (Y_hat - Y_onehot) / n              # (n, 10)  — prediction error

    grads['dW3'] = dZ3.T @ A2                 # (10, 128)
    grads['db3'] = dZ3.sum(axis=0, keepdims=True).T   # (10, 1)

    # --- Hidden layer 2 (ReLU) ---
    dA2 = dZ3 @ params['W3']                  # (n, 128)
    dZ2 = dA2 * relu_derivative(Z2)           # (n, 128)

    grads['dW2'] = dZ2.T @ A1                 # (128, 256)
    grads['db2'] = dZ2.sum(axis=0, keepdims=True).T   # (128, 1)

    # --- Hidden layer 1 (ReLU) ---
    dA1 = dZ2 @ params['W2']                  # (n, 256)
    dZ1 = dA1 * relu_derivative(Z1)           # (n, 256)

    grads['dW1'] = dZ1.T @ A0                 # (256, 784)
    grads['db1'] = dZ1.sum(axis=0, keepdims=True).T   # (256, 1)

    return grads

def update_params(params, grads, lr):
    """SGD parameter update: W = W - lr * dW"""
    updated = {}
    for l in range(1, len(LAYER_SIZES)):
        updated[f'W{l}'] = params[f'W{l}'] - lr * grads[f'dW{l}']
        updated[f'b{l}'] = params[f'b{l}'] - lr * grads[f'db{l}']
    return updated

# --- Gradient check: verify backward pass is correct ---
y_hat_b, cache_b = forward_pass(X_train[:64], params)
grads_b = backward_pass(y_train[:64], cache_b, params)
print("Gradient shapes:")
for k, v in grads_b.items():
    print(f"  {k}: {v.shape}")
The output layer gradient is prediction error. The formula \(\partial\mathcal{L}/\partial Z_3 = (\hat{Y} - Y)/n\) is the combined gradient of softmax + cross-entropy. Cells where the network is most wrong receive the largest gradient signal, driving the biggest weight update.

9. Step 7 — Training Loop

STEP 7 OF 10 • Mini-Batch SGD
def train_network(X_train, y_train, X_val, y_val, params,
                  lr=0.1, epochs=30, batch_size=128):
    """
    Mini-batch SGD training loop.
    Returns: trained params, training history
    """
    n = len(y_train)
    history = {'train_loss': [], 'val_loss': [],
               'train_acc': [],  'val_acc': []}

    for epoch in range(1, epochs + 1):
        # Shuffle training data
        idx = np.random.permutation(n)
        X_shuffled = X_train[idx]
        y_shuffled = y_train[idx]

        epoch_loss = 0.0
        n_batches = 0

        for start in range(0, n, batch_size):
            X_batch = X_shuffled[start:start + batch_size]
            y_batch = y_shuffled[start:start + batch_size]

            # Forward
            y_hat, cache = forward_pass(X_batch, params)

            # Loss
            loss = cross_entropy_loss(y_hat, y_batch)
            epoch_loss += loss
            n_batches += 1

            # Backward
            grads = backward_pass(y_batch, cache, params)

            # Update
            params = update_params(params, grads, lr)

        # --- Record epoch metrics ---
        y_hat_train, _ = forward_pass(X_train, params)
        y_hat_val,   _ = forward_pass(X_val, params)

        train_loss = cross_entropy_loss(y_hat_train, y_train)
        val_loss   = cross_entropy_loss(y_hat_val,   y_val)
        train_acc  = accuracy(y_hat_train, y_train)
        val_acc    = accuracy(y_hat_val,   y_val)

        history['train_loss'].append(train_loss)
        history['val_loss'].append(val_loss)
        history['train_acc'].append(train_acc)
        history['val_acc'].append(val_acc)

        if epoch % 5 == 0 or epoch == 1:
            print(f"Epoch {epoch:3d}/{epochs} | "
                  f"Train Loss: {train_loss:.4f}  Acc: {train_acc:.1%} | "
                  f"Val Loss:   {val_loss:.4f}  Acc: {val_acc:.1%}")

    return params, history

# --- Run training ---
params = initialize_weights(LAYER_SIZES)

print("Training [784 → 256 → 128 → 10] MLP on MNIST...")
print("=" * 75)
params, history = train_network(
    X_train, y_train,
    X_test,  y_test,
    params,
    lr=0.1,
    epochs=30,
    batch_size=128
)
Expected output: You should see training accuracy start around 10% (random guessing with 10 classes) and rise above 95% by epoch 30. If accuracy stays stuck near 10%, there is likely a sign error in your gradient implementation — re-check the output layer gradient formula.

13. Bonus — Numerical Gradient Check

A gradient check verifies that your analytical backprop is correct by comparing it against numerically estimated gradients using finite differences. If your implementation is correct, the relative error should be below \(10^{-5}\).

def numerical_gradient(X, y, params, layer='W1', eps=1e-5):
    """Estimate gradient of W using finite differences (slow — use only for checking)."""
    W = params[layer].copy()
    grad_num = np.zeros_like(W)

    it = np.nditer(W, flags=['multi_index'])
    while not it.finished:
        idx = it.multi_index

        # Loss at W + eps
        params[layer][idx] += eps
        y_hat_plus, _ = forward_pass(X, params)
        loss_plus = cross_entropy_loss(y_hat_plus, y)

        # Loss at W - eps
        params[layer][idx] -= 2 * eps
        y_hat_minus, _ = forward_pass(X, params)
        loss_minus = cross_entropy_loss(y_hat_minus, y)

        # Restore
        params[layer][idx] += eps

        grad_num[idx] = (loss_plus - loss_minus) / (2 * eps)
        it.iternext()

    return grad_num

# Check on a tiny subset (numerical check is slow)
X_check = X_train[:8]
y_check = y_train[:8]
params_check = initialize_weights([784, 4, 4, 10])   # tiny net for speed

y_hat_c, cache_c = forward_pass(X_check, params_check)
grads_c = backward_pass(y_check, cache_c, params_check)

grad_analytical = grads_c['dW1']
grad_numerical  = numerical_gradient(X_check, y_check, params_check, 'W1')

rel_error = (np.abs(grad_analytical - grad_numerical) /
             (np.abs(grad_analytical) + np.abs(grad_numerical) + 1e-10)).max()

print(f"Maximum relative error between analytical and numerical gradient: {rel_error:.2e}")
print("Status:", "PASS ✓" if rel_error < 1e-4 else "FAIL ✗ — check your backward pass")
A relative error below 1e-5 confirms your backpropagation is correct. This test is the gold standard for verifying custom gradient implementations. PyTorch and JAX use automated versions of this check internally.

14. Experiment Ideas

Extend Your Implementation

  • Experiment A — Depth: Add a third hidden layer (e.g., [784 → 512 → 256 → 128 → 10]). Does accuracy improve? How does training time change? Where does the gradient vanish first?
  • Experiment B — Activation Function: Replace ReLU with Sigmoid in the hidden layers. Compare training curves. Observe how quickly accuracy rises compared to ReLU and document the vanishing gradient effect.
  • Experiment C — Learning Rate Schedule: Multiply the learning rate by 0.5 every 10 epochs. Does this smooth the loss curve and improve final accuracy compared to a fixed learning rate?
  • Experiment D — Dropout (Manual): During training, randomly set 20% of hidden layer activations to zero and scale the remaining by 1/0.8. Do not apply dropout at test time. Does test accuracy improve?
  • Experiment E — Momentum: Implement SGD with momentum: \(v \leftarrow \beta v - \eta \nabla W\), \(W \leftarrow W + v\). Use \(\beta = 0.9\). How many fewer epochs are needed to reach 95%?
