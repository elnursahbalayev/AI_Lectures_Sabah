2. The Core Concept

Recall from the lecture: in adversarial environments, we can't just optimize — we must assume the opponent plays optimally too. The Minimax algorithm captures this perfectly:

\[\text{minimax}(S) = \begin{cases} U(S) & \text{if terminal} \\ \max_a \text{minimax}(\text{Result}(S, a)) & \text{MAX's turn} \\ \min_a \text{minimax}(\text{Result}(S, a)) & \text{MIN's turn} \end{cases}\]

MAX (AI — X)

Goal: Maximize the score.

Utility: Win = +1, Draw = 0, Loss = -1

At each turn, MAX picks the child with the highest minimax value.

MIN (Human — O)

Goal: Minimize the score.

Utility: Win = -1, Draw = 0, Loss = +1

At each turn, MIN picks the child with the lowest minimax value.

3. Building the Game Engine

Before we build the brain (AI), we need the body (game engine). We represent a Tic-Tac-Toe board as a list of 9 elements.

Board Representation

# ============================================================
# BOARD REPRESENTATION
# We represent the 3×3 Tic-Tac-Toe board as a flat list of
# 9 elements (indices 0-8).  This makes it easy to copy,
# compare, and iterate over.
#
# Visual mapping of list indices to board positions:
#
#   0 | 1 | 2      ←  top row
#   ---------
#   3 | 4 | 5      ←  middle row
#   ---------
#   6 | 7 | 8      ←  bottom row
#
# Each cell holds one of three values:
#   ' ' (EMPTY)  —  no one has played here yet
#   'X'          —  player X occupies this cell
#   'O'          —  player O occupies this cell
# ============================================================

EMPTY = ' '   # constant for an unoccupied cell
X = 'X'       # constant for player X (the AI)
O = 'O'       # constant for player O (the human)

def create_board():
    """Create a fresh, empty board."""
    # [EMPTY] * 9 creates a list of 9 spaces: [' ',' ',' ',...]
    # This is our starting state — a completely blank board.
    return [EMPTY] * 9

Game Logic

# ============================================================
# WINNING COMBINATIONS
# There are exactly 8 ways to win in Tic-Tac-Toe:
#   3 rows  +  3 columns  +  2 diagonals = 8
# Each sub-list contains the 3 board indices that form a line.
# ============================================================
WIN_LINES = [
    [0, 1, 2], [3, 4, 5], [6, 7, 8],  # 3 horizontal rows
    [0, 3, 6], [1, 4, 7], [2, 5, 8],  # 3 vertical columns
    [0, 4, 8], [2, 4, 6]              # 2 diagonals
]

def check_winner(board):
    """Return 'X', 'O', or None."""
    # Go through each of the 8 possible winning lines
    for line in WIN_LINES:
        a, b, c = line   # unpack the 3 indices of this line

        # Check if all 3 cells in this line are the SAME
        # AND not empty (i.e., someone actually placed there)
        if board[a] == board[b] == board[c] != EMPTY:
            return board[a]   # return 'X' or 'O' — whoever owns the line

    return None   # no winner found yet

def get_available_moves(board):
    """Return list of empty cell indices."""
    # List comprehension: collect every index (0-8) where the
    # cell is still EMPTY.  These are the legal moves.
    # Example: if cells 0,4,8 are taken → returns [1,2,3,5,6,7]
    return [i for i in range(9) if board[i] == EMPTY]

def is_terminal(board):
    """Is the game over?"""
    # The game is over in two cases:
    #   1. Someone has won  (check_winner returns 'X' or 'O')
    #   2. The board is full (no available moves left → draw)
    return check_winner(board) is not None or len(get_available_moves(board)) == 0

def make_move(board, position, player):
    """Return a NEW board with the move applied."""
    # IMPORTANT: We create a COPY of the board so we don't
    # modify the original.  This is essential for recursion —
    # each branch of the game tree needs its own board state.
    new_board = board.copy()        # shallow copy of the list
    new_board[position] = player    # place X or O at the chosen cell
    return new_board                # return the new board state

4. Implementing Minimax

Now for the brain. We translate the mathematical definition directly into Python.

import math   # we need math.inf for positive/negative infinity

# ============================================================
# UTILITY FUNCTION
# Assigns a numeric score to a finished (terminal) game.
# This is the "reward signal" that Minimax uses to decide
# which moves are good and which are bad.
#
#   +1  →  X (the AI) wins    — best outcome for MAX
#    0  →  Draw               — neutral
#   -1  →  O (the human) wins — worst outcome for MAX
# ============================================================
def utility(board):
    """Score a terminal board state."""
    winner = check_winner(board)   # find out who won (or None)
    if winner == X:
        return +1   # AI wins  → positive reward
    elif winner == O:
        return -1   # Human wins → negative reward (bad for AI)
    else:
        return 0    # Draw → neutral

# ============================================================
# MINIMAX ALGORITHM
# Recursively explores the ENTIRE game tree from the current
# board state down to every possible terminal state.
#
# - When it's MAX's turn (AI / X), it picks the move with
#   the HIGHEST value (wants to win).
# - When it's MIN's turn (Human / O), it picks the move with
#   the LOWEST value (wants AI to lose).
#
# Returns: the minimax value of this board state (+1, 0, -1).
# ============================================================
def minimax(board, is_maximizing):
    """Return the minimax value of this board state."""

    # ------- BASE CASE -------
    # If the game is over, return the actual score.
    # This stops the recursion.
    if is_terminal(board):
        return utility(board)

    # ------- MAX's TURN (AI plays X) -------
    if is_maximizing:
        best_value = -math.inf   # start with worst possible for MAX

        # Try every legal move that X can make
        for move in get_available_moves(board):
            child = make_move(board, move, X)   # simulate playing X here
            value = minimax(child, False)        # recurse: now it's MIN's turn
            best_value = max(best_value, value)  # keep the highest score

        return best_value   # the best outcome MAX can guarantee

    # ------- MIN's TURN (Human plays O) -------
    else:
        best_value = math.inf   # start with worst possible for MIN

        # Try every legal move that O can make
        for move in get_available_moves(board):
            child = make_move(board, move, O)   # simulate playing O here
            value = minimax(child, True)         # recurse: now it's MAX's turn
            best_value = min(best_value, value)  # keep the lowest score

        return best_value   # the best outcome MIN can guarantee

# ============================================================
# FIND BEST MOVE
# This is the "decision maker" — the function we actually call
# to get the AI's next move.
#
# It tries every available move for X, runs minimax on each
# resulting board, and returns the move that leads to the
# highest minimax value.
#
# Returns: the index (0-8) of the best cell to play.
# ============================================================
def find_best_move(board):
    """Return the optimal move for X (AI)."""
    best_value = -math.inf   # track the best score seen so far
    best_move = None          # track which move produced that score

    # Try each empty cell
    for move in get_available_moves(board):
        child = make_move(board, move, X)   # simulate: "what if X plays here?"
        value = minimax(child, False)        # evaluate: how does the game end?

        # If this move leads to a better outcome, remember it
        if value > best_value:
            best_value = value   # update best score
            best_move = move     # update best move index

    return best_move   # return the cell index for the optimal move
Key Insight: On an empty board, Minimax explores approximately 549,946 nodes to make the first move. That's the entire game tree — every possible future. For Tic-Tac-Toe, this is fast enough. For Chess, it would take longer than the age of the universe.

🚀 Run It! — Complete Game Loop

Copy all the code from Sections 3 and 4 above into a single .py file, then add the code below at the bottom. This gives you a fully playable terminal game.

# ============================================================
# DISPLAY BOARD
# Prints the board in a nice human-readable 3×3 grid.
# Empty cells show their index number (0-8) so the player
# knows which number to type to make a move.
# ============================================================
def display_board(board):
    """Print the board. Empty cells show their index."""
    print()   # blank line for spacing
    for row in range(3):                      # 3 rows: 0, 1, 2
        cells = []                            # collect cells for this row
        for col in range(3):                  # 3 columns: 0, 1, 2
            idx = row * 3 + col               # convert (row, col) → index 0-8
            if board[idx] == EMPTY:
                cells.append(str(idx))        # show index number for empty cells
            else:
                cells.append(board[idx])      # show 'X' or 'O' for occupied cells
        print(' | '.join(cells))              # join with pipes: "X | 1 | O"
        if row < 2:
            print('---------')                # separator between rows
    print()   # blank line for spacing


# ============================================================
# PLAY GAME — MAIN GAME LOOP
# This function runs a complete game of Tic-Tac-Toe:
#   - Human plays as O, AI plays as X
#   - AI goes first (X always starts in Tic-Tac-Toe)
#   - Loop continues until someone wins or it's a draw
# ============================================================
def play_game():
    """Play a full game: Human (O) vs AI (X)."""
    board = create_board()          # start with an empty board
    print("=== Tic-Tac-Toe: You (O) vs AI (X) ===")
    print("Enter a number 0-8 to place your move.")
    display_board(board)            # show the empty board with indices

    # ------- GAME LOOP -------
    # X always goes first. We alternate between AI and Human.
    current_player = X              # X (AI) starts

    while not is_terminal(board):   # keep playing until game is over

        if current_player == X:
            # ---- AI's TURN ----
            print("AI is thinking...")
            move = find_best_move(board)             # find the optimal move
            board = make_move(board, move, X)         # apply it to the board
            print(f"AI plays position {move}")
        else:
            # ---- HUMAN's TURN ----
            # Keep asking until the player enters a valid move
            while True:
                try:
                    move = int(input("Your move (0-8): "))  # read player input
                    if move < 0 or move > 8:
                        print("Please enter a number between 0 and 8.")
                    elif board[move] != EMPTY:
                        print("That cell is already taken! Try again.")
                    else:
                        break  # valid move — exit the input loop
                except ValueError:
                    print("Invalid input. Please enter a number 0-8.")
            board = make_move(board, move, O)          # apply human's move

        display_board(board)              # show the updated board
        current_player = O if current_player == X else X   # switch turns

    # ------- GAME OVER -------
    winner = check_winner(board)
    if winner == X:
        print("AI wins! Better luck next time.")
    elif winner == O:
        print("You win! (This shouldn't happen against Minimax!)")
    else:
        print("It's a draw! That's the best you can do against a perfect AI.")


# ============================================================
# START THE GAME
# This is the entry point — when you run the script, the
# game starts here.
# ============================================================
play_game()

  3. Append the game loop code above (display_board + play_game)
  4. Run: python tictactoe.py

6. Alpha-Beta Pruning

The Minimax AI works perfectly, but it's wasteful — it explores branches that can never influence the decision. Alpha-Beta Pruning fixes this by maintaining two bounds:

  • \(\alpha\) = best value MAX can guarantee (initially \(-\infty\))
  • \(\beta\) = best value MIN can guarantee (initially \(+\infty\))

Rule: If \(\alpha \geq \beta\), stop searching — the remaining branches are irrelevant. 

# ============================================================
# ALPHA-BETA PRUNING  (optimized Minimax)
# Same logic as Minimax, but we skip branches that we can
# PROVE will never be chosen.  This dramatically reduces
# the number of nodes we need to evaluate.
#
# Parameters:
#   board          - current game state (list of 9 cells)
#   alpha (α)      - best value MAX can guarantee so far
#                    (initially -infinity)
#   beta  (β)      - best value MIN can guarantee so far
#                    (initially +infinity)
#   is_maximizing  - True when it's X's turn, False for O
#
# KEY RULE: If α >= β, stop exploring this branch.
#   It means one player already has a better option
#   elsewhere, so this branch will never be reached
#   in optimal play.
#
# Returns: the minimax value (+1, 0, or -1) — same as
#          plain Minimax, just computed faster.
# ============================================================
def alpha_beta(board, alpha, beta, is_maximizing):
    """Minimax with Alpha-Beta Pruning."""

    # Base case: game is over → return actual score
    if is_terminal(board):
        return utility(board)

    # ------- MAX's TURN (AI / X) -------
    if is_maximizing:
        value = -math.inf   # start with the worst for MAX

        for move in get_available_moves(board):
            child = make_move(board, move, X)   # simulate X playing here

            # Recurse into MIN's turn, passing α and β bounds down
            value = max(value, alpha_beta(child, alpha, beta, False))

            # Update α: MAX now knows it can get at least 'value'
            alpha = max(alpha, value)

            # ✂ PRUNE CHECK (β cutoff):
            # If α >= β, MIN would never allow this path (MIN
            # already has a better option elsewhere scoring ≤ β).
            # So we can safely skip all remaining sibling moves.
            if alpha >= beta:
                break  # β cutoff — prune!
        return value

    # ------- MIN's TURN (Human / O) -------
    else:
        value = math.inf   # start with the worst for MIN

        for move in get_available_moves(board):
            child = make_move(board, move, O)   # simulate O playing here

            # Recurse into MAX's turn, passing α and β bounds down
            value = min(value, alpha_beta(child, alpha, beta, True))

            # Update β: MIN now knows it can hold MAX to at most 'value'
            beta = min(beta, value)

            # ✂ PRUNE CHECK (α cutoff):
            # If α >= β, MAX would never choose this path (MAX
            # already has a guaranteed score of α elsewhere).
            # So we can safely skip all remaining sibling moves.
            if alpha >= beta:
                break  # α cutoff — prune!
        return value

# ============================================================
# FIND BEST MOVE (Alpha-Beta version)
# Same as find_best_move() from Section 4, but uses
# alpha_beta() instead of minimax() for much faster search.
#
# To use this, simply replace the find_best_move() call
# in play_game() with find_best_move_ab().
# ============================================================
def find_best_move_ab(board):
    """Return the optimal move for X using Alpha-Beta Pruning."""
    best_value = -math.inf   # track the best score so far
    best_move = None          # track which move produced it

    for move in get_available_moves(board):
        child = make_move(board, move, X)   # simulate X playing here

        # Use alpha_beta instead of minimax — same result, fewer nodes!
        # Initial α = -infinity, β = +infinity (no bounds yet)
        value = alpha_beta(child, -math.inf, math.inf, False)

        if value > best_value:
            best_value = value
            best_move = move

    return best_move   # same optimal move, computed much faster
To use Alpha-Beta in your game: In the play_game() function, simply change find_best_move(board) to find_best_move_ab(board). The AI will make the exact same moves — but explore far fewer nodes.
Why does pruning not change the result?

Consider this scenario: MAX has already found a move yielding value 3 (\(\alpha = 3\)). While exploring another branch, MIN finds a child with value 2. Since MIN will pick at most 2 (minimizing), and MAX already has 3, MAX would never choose this branch. The remaining children cannot produce a value higher than 2 for MAX (because MIN controls this subtree), so they're irrelevant.

The pruned result is mathematically identical to full Minimax — we just skip provably useless work.
