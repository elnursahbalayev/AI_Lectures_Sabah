0. Lecture Roadmap

Every modern AI system — from image classifiers to ChatGPT — is built on a surprisingly simple idea: stack layers of weighted sums and nonlinear squashing functions, then use calculus to adjust the weights until the output is correct. Understanding this mechanism is the most important conceptual step in the entire course.

In Weeks 1–8, we built classical ML models that are powerful but limited — they cannot automatically learn hierarchical features from raw data like pixels or text. Neural networks change that. Today we build the mathematical and conceptual foundation for everything in Module 3.

1. The Biological Analogy

A biological neuron fires an electrical signal if the total input from connected neurons exceeds a threshold. An artificial neuron is a mathematical abstraction of this idea:

x₁ ——[w₁]——┐
x₂ ——[w₂]——┤→ z = w₁x₁ + w₂x₂ + w₃x₃ + b → a = σ(z)
x₃ ——[w₃]——┘

Where:

  • \(x_i\) — input features
  • \(w_i\) — weights (learned during training)
  • \(b\) — bias (a learned offset, like the intercept in linear regression)
  • \(z\) — the pre-activation (linear combination)
  • \(\sigma\) — the activation function (nonlinear squashing)
  • \(a\) — the activation output passed to the next layer
$$z = \mathbf{w}^\top \mathbf{x} + b \qquad a = \sigma(z)$$
Key insight: Without the activation function \(\sigma\), stacking multiple layers is equivalent to a single linear transformation. The nonlinearity is what makes deep networks capable of learning complex, non-linear patterns.

1.1 From One Neuron to One Layer

A layer contains multiple neurons processing the same input in parallel. If layer \(\ell\) has \(n\) neurons each receiving \(d\) inputs, we can express the entire layer as a single matrix operation:

$$\mathbf{z}^{(\ell)} = W^{(\ell)} \mathbf{a}^{(\ell-1)} + \mathbf{b}^{(\ell)}$$ $$\mathbf{a}^{(\ell)} = \sigma\!\left(\mathbf{z}^{(\ell)}\right)$$

Where \(W^{(\ell)} \in \mathbb{R}^{n \times d}\) is the weight matrix for layer \(\ell\). This matrix-vector product simultaneously computes all \(n\) neurons' pre-activations in a single GPU-accelerated operation.

2. The Multi-Layer Perceptron (MLP)

An MLP chains multiple layers together. Each layer transforms the output of the previous one:

Input Layer (raw features)
↓
Hidden Layer 1 (learned features)
↓
Hidden Layer 2 (more abstract features)
↓
Output Layer (predictions)
Layer Role Typical Activation
Input Layer Receives raw features — no computation, just passes data through None
Hidden Layers Learn intermediate representations; stacking gives depth ReLU (default)
Output Layer Produces the final prediction — regression or classification Linear / Sigmoid / Softmax

2.1 Network Notation

An MLP is fully described by its architecture: [784 → 256 → 128 → 10] means an input of 784 features, two hidden layers (256 and 128 neurons), and an output of 10 classes. The total parameter count is:

$$\text{Parameters} = \sum_{\ell=1}^{L} \left( n_{\ell} \times n_{\ell-1} + n_{\ell} \right)$$

For [784 → 256 → 128 → 10]: (784×256 + 256) + (256×128 + 128) + (128×10 + 10) = 234,826 parameters. All of these are learned by gradient descent.

Why "deep" learning? Depth (many layers) allows a network to learn hierarchical representations. A CNN trained on faces learns: edges → curves → eyes/nose → faces. Each layer builds on abstractions from the layer below. This hierarchy is impossible with a single layer.

3. Activation Functions

The activation function is what separates a neural network from a stack of linear regressions. Without it, any depth collapses to a single matrix multiply.

Sigmoid
σ(z) = 1 / (1 + e⁻ᶻ)
Output: (0, 1). Binary classification output layer. Historically used in hidden layers, now largely replaced by ReLU.
Tanh
tanh(z) = (eᶻ − e⁻ᶻ) / (eᶻ + e⁻ᶻ)
Output: (−1, 1). Zero-centered (better than sigmoid for hidden layers). Still suffers vanishing gradients at extremes.
ReLU
ReLU(z) = max(0, z)
Default choice for hidden layers. Cheap to compute, avoids vanishing gradient for positive inputs. Risk: "dying ReLU" for large negative inputs.
Softmax
softmax(zᵢ) = eᶻⁱ / Σⱼ eᶻʲ
Output: probability distribution over K classes. Always used in the output layer for multi-class classification. All outputs sum to 1.

3.1 The Vanishing Gradient Problem

Sigmoid and Tanh saturate: when \(|z|\) is large, the derivative \(\sigma'(z) \approx 0\). During backpropagation, gradients are multiplied through many layers. If each derivative is a small number, the product approaches zero — the gradient vanishes before reaching the early layers, which then receive essentially no learning signal.

$$\sigma'(z) = \sigma(z)(1 - \sigma(z)) \quad \text{max value: } 0.25 \text{ at } z=0$$
ReLU avoids this. Its derivative is exactly 1 for all positive inputs — gradients pass through unchanged. This is the primary reason ReLU became the default activation in deep networks after 2012.

3.2 Derivatives of Activation Functions

FunctionDerivativeRange of Derivative
Sigmoid \(\sigma\)\(\sigma(z)(1-\sigma(z))\)\([0, 0.25]\)
Tanh\(1 - \tanh^2(z)\)\([0, 1]\)
ReLU\(1\) if \(z > 0\), else \(0\)\(\{0, 1\}\)
Softmax \(s_i\)\(s_i(1 - s_i)\) (diagonal of Jacobian)\([0, 0.25]\)

4. The Forward Pass

The forward pass is the process of computing a prediction from an input. For a 3-layer MLP [input → hidden₁ → hidden₂ → output]:

$$\mathbf{a}^{(0)} = \mathbf{x}$$ $$\mathbf{z}^{(1)} = W^{(1)}\mathbf{a}^{(0)} + \mathbf{b}^{(1)}, \quad \mathbf{a}^{(1)} = \text{ReLU}(\mathbf{z}^{(1)})$$ $$\mathbf{z}^{(2)} = W^{(2)}\mathbf{a}^{(1)} + \mathbf{b}^{(2)}, \quad \mathbf{a}^{(2)} = \text{ReLU}(\mathbf{z}^{(2)})$$ $$\mathbf{z}^{(3)} = W^{(3)}\mathbf{a}^{(2)} + \mathbf{b}^{(3)}, \quad \hat{\mathbf{y}} = \text{softmax}(\mathbf{z}^{(3)})$$

The output \(\hat{\mathbf{y}}\) is a probability vector over all classes. In NumPy, a single forward pass for one sample looks like:

import numpy as np

def relu(z):
    return np.maximum(0, z)

def softmax(z):
    exp_z = np.exp(z - z.max())   # subtract max for numerical stability
    return exp_z / exp_z.sum()

def forward(x, params):
    W1, b1, W2, b2, W3, b3 = params

    z1 = W1 @ x + b1
    a1 = relu(z1)

    z2 = W2 @ a1 + b2
    a2 = relu(z2)

    z3 = W3 @ a2 + b3
    y_hat = softmax(z3)

    # Cache for backprop
    cache = (x, z1, a1, z2, a2, z3, y_hat)
    return y_hat, cache
Why subtract max in softmax? \(e^{z}\) overflows for large \(z\) (e.g., \(e^{1000} = \infty\) in float32). Subtracting the maximum value before exponentiation is mathematically equivalent but numerically safe — this is called the log-sum-exp trick.

5. Loss Functions

A loss function quantifies how wrong the network's prediction is. Training is the process of minimizing this function by adjusting weights.

5.1 Mean Squared Error (Regression)

$$\mathcal{L}_{\text{MSE}} = \frac{1}{n}\sum_{i=1}^{n}(y_i - \hat{y}_i)^2$$

5.2 Binary Cross-Entropy (Binary Classification)

$$\mathcal{L}_{\text{BCE}} = -\frac{1}{n}\sum_{i=1}^{n}\left[y_i \log \hat{y}_i + (1-y_i)\log(1-\hat{y}_i)\right]$$

5.3 Categorical Cross-Entropy (Multi-class Classification)

$$\mathcal{L}_{\text{CE}} = -\frac{1}{n}\sum_{i=1}^{n}\sum_{k=1}^{K} y_{ik} \log \hat{y}_{ik}$$

Since only one class is correct per sample (\(y_{ik} = 1\) for the true class, 0 elsewhere), this simplifies to:

$$\mathcal{L}_{\text{CE}} = -\frac{1}{n}\sum_{i=1}^{n} \log \hat{y}_{i, \text{true}}$$ 
def cross_entropy_loss(y_hat, y_true):
    """
    y_hat: (n, K) probability matrix from softmax
    y_true: (n,) integer class labels
    """
    n = len(y_true)
    # Pick the predicted probability for the correct class
    correct_class_probs = y_hat[np.arange(n), y_true]
    loss = -np.mean(np.log(correct_class_probs + 1e-9))   # 1e-9 avoids log(0)
    return loss
Why cross-entropy for classification? Cross-entropy is derived from Maximum Likelihood Estimation applied to a categorical distribution. It penalizes confident wrong predictions extremely heavily (log(~0) → −∞), which drives the model to assign high probability to the correct class.

6. Gradient Descent — The Optimization Loop

Training a neural network is an optimization problem: find weights \(W\) that minimize \(\mathcal{L}(W)\). We do this with gradient descent — repeatedly step in the direction of steepest descent.

$$W \leftarrow W - \eta \cdot \frac{\partial \mathcal{L}}{\partial W}$$

Where \(\eta\) is the learning rate — arguably the most important hyperparameter in deep learning.

VariantUpdate UsesTrade-off
Batch GD All training samples Stable gradients, but slow on large datasets
Stochastic GD (SGD) One sample at a time Fast updates, noisy gradients, can escape local minima
Mini-Batch GD A batch (e.g., 32–256 samples) Best of both — standard practice in deep learning
Mini-batch is the standard. Batch sizes of 32–256 provide stable gradient estimates while being small enough for fast iteration. They also exploit GPU parallelism efficiently — a single matrix multiply on a batch of 256 samples is nearly as fast as one on a single sample.

7. Backpropagation — The Chain Rule Applied

Backpropagation is the algorithm that computes \(\partial \mathcal{L} / \partial W^{(\ell)}\) for every layer efficiently. It is simply the Chain Rule from calculus applied systematically from output to input.

7.1 The Chain Rule (Scalar Review)

If \(y = f(g(x))\), then:

$$\frac{dy}{dx} = \frac{dy}{dg} \cdot \frac{dg}{dx}$$

7.2 One Layer — Deriving the Gradients

Consider the output layer with softmax + cross-entropy. The gradient of the loss with respect to the pre-activation \(\mathbf{z}^{(L)}\) has an elegant closed form:

$$\frac{\partial \mathcal{L}}{\partial \mathbf{z}^{(L)}} = \hat{\mathbf{y}} - \mathbf{y} \qquad \text{(softmax + cross-entropy)}$$

This is one of the most important results in deep learning: the gradient at the output layer is simply the prediction error. From here, we propagate backward through each layer:

$$\frac{\partial \mathcal{L}}{\partial W^{(\ell)}} = \frac{\partial \mathcal{L}}{\partial \mathbf{z}^{(\ell)}} \cdot \left(\mathbf{a}^{(\ell-1)}\right)^\top$$ $$\frac{\partial \mathcal{L}}{\partial \mathbf{b}^{(\ell)}} = \frac{\partial \mathcal{L}}{\partial \mathbf{z}^{(\ell)}}$$ $$\frac{\partial \mathcal{L}}{\partial \mathbf{a}^{(\ell-1)}} = \left(W^{(\ell)}\right)^\top \cdot \frac{\partial \mathcal{L}}{\partial \mathbf{z}^{(\ell)}}$$

To propagate the gradient back through the ReLU activation:

$$\frac{\partial \mathcal{L}}{\partial \mathbf{z}^{(\ell)}} = \frac{\partial \mathcal{L}}{\partial \mathbf{a}^{(\ell)}} \odot \mathbf{1}\left[\mathbf{z}^{(\ell)} > 0\right]$$

The symbol \(\odot\) denotes element-wise multiplication. The indicator \(\mathbf{1}[z > 0]\) is the derivative of ReLU — it simply zeros out gradients where the pre-activation was negative.

7.3 Full Backpropagation in NumPy

def backward(y_hat, y_true, cache, params):
    x, z1, a1, z2, a2, z3, _ = cache
    W1, b1, W2, b2, W3, b3 = params
    n = len(y_true)

    # One-hot encode true labels
    Y = np.zeros_like(y_hat)
    Y[np.arange(n), y_true] = 1

    # --- Output layer gradient (softmax + cross-entropy) ---
    dz3 = (y_hat - Y) / n           # (n, 10)

    dW3 = a2.T @ dz3                # (128, 10)
    db3 = dz3.sum(axis=0)           # (10,)

    # --- Hidden layer 2 ---
    da2 = dz3 @ W3.T                # (n, 128)
    dz2 = da2 * (z2 > 0)            # ReLU derivative

    dW2 = a1.T @ dz2                # (256, 128)
    db2 = dz2.sum(axis=0)           # (128,)

    # --- Hidden layer 1 ---
    da1 = dz2 @ W2.T                # (n, 256)
    dz1 = da1 * (z1 > 0)            # ReLU derivative

    dW1 = x.T @ dz1                 # (784, 256)
    db1 = dz1.sum(axis=0)           # (256,)

    grads = (dW1, db1, dW2, db2, dW3, db3)
    return grads
Key observation: The gradient for each layer's weights is the outer product of the upstream error signal and the layer's input activations. Intuitively: a weight connecting a large activation to a large error gets a large gradient update.

7.4 The Weight Update Step

def update_params(params, grads, learning_rate):
    updated = []
    for param, grad in zip(params, grads):
        updated.append(param - learning_rate * grad)
    return tuple(updated)

8. The Complete Training Loop

The full training loop puts all pieces together. Each epoch is one complete pass over the training data:

def train(X_train, y_train, params, learning_rate=0.01, epochs=50, batch_size=64):
    n = len(y_train)
    history = []

    for epoch in range(epochs):
        # Shuffle training data each epoch
        idx = np.random.permutation(n)
        X_train = X_train[idx]
        y_train = y_train[idx]

        epoch_loss = 0
        n_batches = 0

        # Mini-batch loop
        for start in range(0, n, batch_size):
            X_batch = X_train[start:start + batch_size]
            y_batch = y_train[start:start + batch_size]

            # Forward pass
            y_hat, cache = forward(X_batch, params)

            # Compute loss
            loss = cross_entropy_loss(y_hat, y_batch)
            epoch_loss += loss
            n_batches += 1

            # Backward pass
            grads = backward(y_hat, y_batch, cache, params)

            # Update weights
            params = update_params(params, grads, learning_rate)

        avg_loss = epoch_loss / n_batches
        history.append(avg_loss)

        if (epoch + 1) % 10 == 0:
            acc = accuracy(X_train, y_train, params)
            print(f"Epoch {epoch+1:3d}/{epochs} | Loss: {avg_loss:.4f} | Train Acc: {acc:.1%}")

    return params, history
For each epoch:
  Shuffle data → split into mini-batches
  For each mini-batch:
    Forward Pass → compute ŷ
    Compute Loss L(ŷ, y)
    Backward Pass → compute ∂L/∂W for all layers
    Update: W ← W − η · ∂L/∂W

9. Practical Considerations

9.1 Weight Initialization

Do not initialize all weights to zero — every neuron in a layer would compute the same gradient and update identically, a problem called symmetry breaking failure. Instead:

InitializerFormulaBest For
Random Normal \(\mathcal{N}(0, 0.01)\) Quick experiments, small networks
Xavier / Glorot \(\mathcal{N}\!\left(0, \sqrt{\frac{2}{n_{in}+n_{out}}}\right)\) Sigmoid / Tanh activations
He (Kaiming) \(\mathcal{N}\!\left(0, \sqrt{\frac{2}{n_{in}}}\right)\) ReLU activations — default choice 
def initialize_params(layer_sizes):
    """He initialization for ReLU networks."""
    params = []
    for i in range(len(layer_sizes) - 1):
        n_in  = layer_sizes[i]
        n_out = layer_sizes[i + 1]
        W = np.random.randn(n_out, n_in) * np.sqrt(2.0 / n_in)   # He init
        b = np.zeros(n_out)
        params.extend([W, b])
    return tuple(params)

9.2 Learning Rate

The learning rate \(\eta\) is the most sensitive hyperparameter:

  • Too large: loss oscillates or diverges — the update step overshoots the minimum.
  • Too small: training is painfully slow — thousands of epochs for convergence.
  • Just right: loss decreases smoothly and plateaus near a minimum.
Rule of thumb: Start with \(\eta = 0.01\) for plain SGD. Use a learning rate schedule or optimizer (Adam, RMSProp) for production training — these adapt the learning rate automatically per parameter.

11. Summary

  1. An artificial neuron computes a weighted sum of inputs plus a bias, then applies a nonlinear activation function.
  2. An MLP stacks layers of neurons; depth enables hierarchical feature learning impossible with a single linear model.
  3. Activation functions introduce nonlinearity. ReLU is the default for hidden layers; Softmax for multi-class output.
  4. The Forward Pass propagates input through all layers using matrix multiplications and activation functions.
  5. Cross-Entropy Loss measures prediction error for classification — derived from maximum likelihood, penalizes confident wrong predictions heavily.
  6. Gradient Descent minimizes the loss by stepping in the negative gradient direction with step size \(\eta\).
  7. Backpropagation is the Chain Rule applied layer-by-layer from output to input, computing \(\partial \mathcal{L}/\partial W\) for every layer.
  8. The output layer gradient for softmax + cross-entropy is simply \(\hat{\mathbf{y}} - \mathbf{y}\) — the prediction error.
  9. He initialization sets weights proportional to \(\sqrt{2/n_{in}}\) to prevent vanishing/exploding activations with ReLU.
  10. Mini-batch gradient descent is the standard: batches of 32–256 balance gradient quality, training speed, and GPU utilization.
