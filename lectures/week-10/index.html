0. Lecture Roadmap

In 2012, a CNN called AlexNet cut the ImageNet error rate nearly in half — from 26% to 15% — defeating every non-deep-learning system by a margin so large it permanently ended the era of hand-crafted vision features. Understanding why requires understanding convolutions.

1. Why MLPs Fail on Images

1.1 The Curse of Dimensionality

A 224×224 RGB image contains \(224 \times 224 \times 3 = 150{,}528\) pixels. A single fully-connected hidden layer of 1,024 neurons would require \(150{,}528 \times 1{,}024 \approx 154\) million parameters — just in the first layer. This is computationally wasteful and prone to overfitting.

But the deeper problem is structural. An MLP treats pixel (0,0) and pixel (223,223) as completely independent features. It ignores three fundamental properties of natural images:

PropertyMeaningMLP Handles It?
Local Correlation Nearby pixels are statistically related — edges are local No — treats all pixels independently
Translation Invariance A cat in the top-left is the same cat in the bottom-right No — different weights for every position
Compositional Hierarchy Edges → textures → parts → objects Partially — but without spatial sharing
CNNs were designed to exploit all three properties. The convolution operation shares weights across all spatial positions, looks at local neighborhoods, and stacks layers to build hierarchical representations — solving all three problems at once.

2. The Convolution Operation

2.1 What a Convolution Does

A convolution slides a small matrix called a kernel (or filter) across the input image and computes the dot product at each position. The result is a feature map that highlights wherever the pattern encoded by the kernel appears in the input.

Input (5×5) ──[slide 3×3 kernel]──→ Feature Map (3×3)

At each position: output = Σ (kernel × input_patch) + bias

2.3 Formal Definition

$$(I * K)[i,j] = \sum_{m=0}^{k_H-1} \sum_{n=0}^{k_W-1} I[i+m,\; j+n] \cdot K[m,n] + b$$

Where \(I\) is the input, \(K\) is the kernel of size \(k_H \times k_W\), and \(b\) is the bias. In deep learning, the kernel weights \(K\) are learned — the network discovers which patterns to detect automatically.

2.4 Stride

Stride controls how many pixels the kernel moves at each step. Stride 1 moves one pixel at a time (maximum resolution); stride 2 skips every other position (halves the output size).

StrideEffectUse Case
s = 1Dense feature map, preserves resolutionMost hidden conv layers
s = 2Halves spatial dimensions, downsamplesReplace pooling (e.g., ResNet)

2.5 Padding

Without padding, each convolution shrinks the feature map. After many layers, the spatial dimensions collapse. Padding adds zeros around the border to control output size:

PaddingPyTorch argumentEffect
Valid (no padding)padding=0Output is smaller than input — spatial info lost at edges
Same (zero padding)padding=k//2Output has the same H×W as input — standard choice

3. Output Size Formula

Given an input of size \(H \times W\), a kernel of size \(k \times k\), padding \(p\), and stride \(s\):

$$H_{out} = \left\lfloor \frac{H + 2p - k}{s} \right\rfloor + 1 \qquad W_{out} = \left\lfloor \frac{W + 2p - k}{s} \right\rfloor + 1$$
Input HKernel kPadding pStride sOutput H
32311\(\lfloor(32+2-3)/1\rfloor+1 = 32\) (same)
32301\(\lfloor(32+0-3)/1\rfloor+1 = 30\) (shrinks)
32312\(\lfloor(32+2-3)/2\rfloor+1 = 16\) (halves)
224732\(\lfloor(224+6-7)/2\rfloor+1 = 112\) (ResNet first layer)

4. Multiple Channels and Multiple Filters

4.1 Input with C Channels

An RGB image has 3 channels. A kernel for a 3-channel input has shape \(k \times k \times C_{in}\) — it slides across all channels simultaneously and sums the results:

$$(I * K)[i,j] = \sum_{c=1}^{C_{in}} \sum_{m=0}^{k-1} \sum_{n=0}^{k-1} I_c[i+m,\,j+n] \cdot K_c[m,n] + b$$

4.2 Multiple Filters → Multiple Feature Maps

Each filter detects one pattern. To detect \(C_{out}\) different patterns simultaneously, we apply \(C_{out}\) filters in parallel, each producing one feature map. The output has shape \(C_{out} \times H_{out} \times W_{out}\).

Parameter count for one conv layer:
\(\text{Params} = C_{out} \times (k \times k \times C_{in} + 1)\)

Example: Conv2d(3, 64, kernel_size=3, padding=1) has \(64 \times (3 \times 3 \times 3 + 1) = 1{,}792\) parameters — compared to 150M for a fully-connected layer on the same 224×224 input.

5. Pooling Layers

Pooling reduces spatial dimensions while retaining the most important information. It provides spatial invariance — a detected feature remains active even if it shifts by a few pixels.

TypeOperationUse
Max Pooling Take the maximum value in each window Most common — retains the strongest feature activation
Average Pooling Take the average value in each window Smoother downsampling; used in GoogLeNet and Global Average Pooling
Global Average Pooling Average across the entire spatial dimension → scalar per channel Replaces the flatten + FC layer in modern architectures (ResNet)

A 2×2 max pool with stride 2 halves both spatial dimensions: a 28×28 feature map becomes 14×14. There are no learnable parameters in a pooling layer.

import torch.nn as nn

# Max pool: 2×2 window, stride 2 → halves H and W
max_pool = nn.MaxPool2d(kernel_size=2, stride=2)

# Global Average Pooling → one value per channel
gap = nn.AdaptiveAvgPool2d(output_size=(1, 1))

6. The Full CNN Architecture

A typical CNN alternates convolutional blocks with pooling, then flattens and passes through fully-connected layers for classification:

Input (3 × H × W)
↓
Conv Block 1: Conv2d → BatchNorm → ReLU → MaxPool
↓
Conv Block 2: Conv2d → BatchNorm → ReLU → MaxPool
↓
Conv Block 3: Conv2d → BatchNorm → ReLU
↓
Global Average Pooling
↓
Fully Connected → Softmax (K classes)

6.1 Batch Normalization

BatchNorm normalizes each feature map to zero mean and unit variance across the batch, then applies learnable scale and shift parameters:

$$\hat{x} = \frac{x - \mu_B}{\sqrt{\sigma_B^2 + \epsilon}} \qquad y = \gamma \hat{x} + \beta$$

Benefits: speeds up training (allows higher learning rates), reduces sensitivity to initialization, acts as mild regularization.

BatchNorm placement: Convention is Conv → BatchNorm → ReLU. BatchNorm is applied before activation because it normalizes the pre-activation values, keeping them in the range where ReLU is most informative.

7. Landmark CNN Architectures

LeNet-5 (1998)

LeCun et al. First successful CNN. 2 conv layers + 3 FC layers. Trained on MNIST. Proved the concept — but hardware limited scaling for 14 years.

AlexNet (2012)

Won ImageNet by 10 percentage points. 5 conv layers, ReLU activations, Dropout, GPU training. The paper that launched the deep learning era.

VGG-16/19 (2014)

Very simple rule: use only 3×3 kernels, always. 16–19 layers deep. Easy to understand and modify — still a strong baseline for transfer learning.

ResNet (2015)

Introduced residual (skip) connections. Trained networks 50–152 layers deep. Won every major ImageNet challenge. Default backbone for most vision tasks today.

7.1 Why Deeper Networks Stopped Working — Before ResNet

In 2015, it was well-known that adding more layers hurt accuracy — not just from overfitting, but even on the training set. A 56-layer network performed worse than a 20-layer network on training data. This was the degradation problem.

The root cause: gradients vanish over 50+ layers of standard convolutions. By the time the gradient reaches the early layers, it is effectively zero — those layers stop learning entirely.

8. Residual Connections — Why Deep Networks Finally Worked

The key insight of ResNet (He et al., 2015): instead of learning a direct mapping \(H(x)\), let each block learn a residual \(F(x) = H(x) - x\). Then the output is:

$$\text{output} = F(\mathbf{x}) + \mathbf{x}$$

This skip connection (or shortcut connection) bypasses the conv layers entirely and adds the input directly to the output:

Input x
├── Conv → BN → ReLU → Conv → BN ──→ F(x)
└──────────────────────────────────→ x (skip)
↓
Output: F(x) + x → ReLU

8.1 Why Does This Work?

  • Gradient highways: The skip connection provides a direct path for gradients to flow back to early layers, bypassing the potentially vanishing-gradient-prone conv stack.
  • Identity shortcut: If the optimal transformation is the identity, the conv layers simply learn \(F(x) \approx 0\). This is much easier than forcing a layer to learn the identity from scratch.
  • Ensemble effect: Residual networks can be interpreted as an implicit ensemble of many shallower networks of varying depth.
import torch
import torch.nn as nn

class ResidualBlock(nn.Module):
    def __init__(self, channels):
        super().__init__()
        self.conv1 = nn.Conv2d(channels, channels, 3, padding=1, bias=False)
        self.bn1   = nn.BatchNorm2d(channels)
        self.conv2 = nn.Conv2d(channels, channels, 3, padding=1, bias=False)
        self.bn2   = nn.BatchNorm2d(channels)
        self.relu  = nn.ReLU(inplace=True)

    def forward(self, x):
        residual = x                        # save input (the skip connection)

        out = self.relu(self.bn1(self.conv1(x)))
        out = self.bn2(self.conv2(out))     # F(x)

        out = out + residual                # F(x) + x
        return self.relu(out)               # apply activation AFTER addition
ResNet results (ILSVRC 2015): A 152-layer ResNet achieved 3.57% top-5 error on ImageNet — better than average human performance (5.1%). The winner used an ensemble of 6 such networks.

9. Transfer Learning

Training a ResNet-50 on ImageNet from scratch requires ~1.2 million labeled images, weeks of GPU time, and significant expertise. Most practitioners never do this. Instead, they use transfer learning: start from a pretrained network and adapt it to a new task.

The filters learned on ImageNet — edges, textures, color gradients, object parts — are useful for almost any visual recognition task. You don't need a million images; you need a smart starting point.

9.1 Why Transfer Learning Works

The lower layers of any image CNN learn universal features:

  • Layer 1–2: Edge detectors, color blobs, Gabor-like filters — identical across datasets.
  • Layer 3–5: Textures, corners, curves — general enough to transfer.
  • Layer 6+: Object-specific parts (wheels, eyes, fur) — increasingly task-specific.
  • Final layer: ImageNet class scores — completely task-specific, always replaced.

9.2 Two Transfer Learning Strategies

StrategyFrozen LayersBest For
Feature Extraction All layers frozen — only the new head is trained Very small datasets (<1,000 images), similar domain to ImageNet
Full Fine-Tuning All layers unfrozen — entire network trained at a low learning rate Medium-large datasets, or domain very different from ImageNet
Gradual Unfreezing Unfreeze from top down, train each stage separately Best practice for limited data — popularized by fast.ai 
import torch
import torchvision.models as models
import torch.nn as nn

# Load ResNet-18 pretrained on ImageNet
model = models.resnet18(weights='IMAGENET1K_V1')

# --- Strategy 1: Feature Extraction (freeze backbone) ---
for param in model.parameters():
    param.requires_grad = False

# Replace the final FC layer for binary classification
model.fc = nn.Linear(model.fc.in_features, 2)   # only this layer trains

# --- Strategy 2: Full Fine-Tuning ---
for param in model.parameters():
    param.requires_grad = True    # unfreeze everything

model.fc = nn.Linear(model.fc.in_features, 2)

# Use a lower learning rate for the backbone than the head
optimizer = torch.optim.Adam([
    {'params': model.fc.parameters(),          'lr': 1e-3},
    {'params': [p for n, p in model.named_parameters() if 'fc' not in n], 'lr': 1e-5}
])
Always fine-tune with a lower learning rate. The pretrained weights are already good — large updates will destroy the useful features. A learning rate 10–100× smaller than normal (e.g., 1e-5 instead of 1e-3) is typical for fine-tuning.

9.3 Data Augmentation for Transfer Learning

from torchvision import transforms

# Training augmentation
train_transform = transforms.Compose([
    transforms.Resize((256, 256)),
    transforms.RandomCrop(224),
    transforms.RandomHorizontalFlip(p=0.5),
    transforms.ColorJitter(brightness=0.2, contrast=0.2, saturation=0.2),
    transforms.ToTensor(),
    transforms.Normalize(mean=[0.485, 0.456, 0.406],   # ImageNet mean
                         std=[0.229, 0.224, 0.225])     # ImageNet std
])

# Validation / Test — no augmentation, just resize and center crop
val_transform = transforms.Compose([
    transforms.Resize((256, 256)),
    transforms.CenterCrop(224),
    transforms.ToTensor(),
    transforms.Normalize(mean=[0.485, 0.456, 0.406],
                         std=[0.229, 0.224, 0.225])
])
Always normalize with ImageNet statistics when fine-tuning a pretrained model. The pretrained weights were learned on data normalized this way — using different statistics shifts all the input distributions and partially invalidates the pretrained features.

11. Summary

  1. MLPs fail on images because they ignore spatial structure, require enormous parameter counts, and have no translation invariance.
  2. Convolution slides a learnable kernel across the image, computing dot products at each position — detecting the pattern encoded in the kernel everywhere it appears.
  3. Stride controls how fast the kernel moves; padding controls whether the output shrinks.
  4. Multiple filters detect multiple patterns in parallel, producing \(C_{out}\) feature maps per layer.
  5. Pooling reduces spatial dimensions, increasing translation invariance — no learnable parameters.
  6. BatchNorm normalizes activations, enabling higher learning rates and faster convergence.
  7. VGG showed that stacking 3×3 convolutions is sufficient for high accuracy; depth matters more than large kernels.
  8. Residual connections (ResNet) solved the degradation problem by providing gradient highways and making the identity mapping easy to learn.
  9. Transfer learning repurposes ImageNet-pretrained features for new tasks — achieves high accuracy with far fewer images and training time.
  10. Always normalize with ImageNet statistics and use a low learning rate when fine-tuning a pretrained CNN.
